# 1. Introduction and Reading Files

Up to this point, all the data we have processed in our Python programs has been stored in **RAM** (Random Access Memory). The moment your script finishes executing or your computer turns off, that data is lost forever. 

To make our data permanent, we must save it to non-volatile storage (like an SSD or Hard Drive) in the form of **Files**.

---

## 1. Text Files vs. Binary Files

All files stored on your disk are ultimately blocks of binary data (`0`s and `1`s). However, in Python, we categorize files into two primary types:

```text
       ┌────────────────────────────────────────────────────────┐
       │                       FILE TYPES                       │
       └────────────────────────────────────────────────────────┘
            │                                                │
     ┌──────┴──────┐                                  ┌──────┴──────┐
     │ Text Files  │                                  │Binary Files │
     ├─────────────┤                                  ├─────────────┤
     │- Characters │                                  │- Raw bytes  │
     │  (ASCII/UTF)│                                  │  (unencoded)│
     │- Human-     │                                  │- Needs special
     │  readable   │                                  │  software   │
     │- .txt, .py  │                                  │- .png, .mp3 │
     └─────────────┘                                  └─────────────┘
```

---

## 2. The Traditional File Workflow: Open, Read/Write, Close

In Python, the traditional way to interact with a file involves three steps:
1. **Open** the file using the built-in `open()` function, which returns a file object.
2. **Perform** operations (read or write).
3. **Close** the file using the `.close()` method to release system resources.

### The Syntax of `open()`
```python
file_object = open(file_name, mode)
```
- `file_name`: The path to the file on your disk (absolute or relative).
- `mode`: A string indicating what you plan to do (e.g., `'r'` for reading, `'w'` for writing).

> [!NOTE]
> **File Object Type:**
> The object returned by the `open()` function depends on the mode:
> - In **Text Mode** (default, e.g. `'r'`, `'w'`), it returns a **`_io.TextIOWrapper`** object.
> - In **Binary Mode** (e.g. `'rb'`, `'wb'`), it returns a **`_io.BufferedReader`** or **`_io.BufferedWriter`** object.

> [!WARNING]
> **Why closing files is critical:**
> Operating systems limit the number of files a program can open simultaneously. If you forget to call `.close()`, you can cause **resource leaks**, lock files from other programs, or lose buffered data that hasn't been written to the disk yet.

---

## 3. The Modern Solution: The `with` Statement (Context Manager)

To prevent resource leaks and avoid having to write `.close()` manually every time, Python provides the **`with` statement** (also known as a **Context Manager**).

This is the industry standard for file handling. It guarantees that the file is closed automatically as soon as the block of code inside the `with` statement finishes executing—**even if an exception or error occurs** inside the block.

```python
with open("settings.txt", "r") as file:
    # Perform file operations here
    pass
# File is automatically closed here!
```

---

## 4. Reading Files in Python

Let's assume we have a configuration file named `settings.txt` containing the following lines:
```text
theme=dark
auto_save=true
font_size=14
```

Python provides four primary ways to read this data:

### 1. `.read(size)`
Reads the specified number of characters from the file. If no size is specified, it reads the **entire** file into a single string.

**Input:**
```python
# Assuming 'settings.txt' contains the lines above
with open("settings.txt", "r") as file:
    content = file.read()
    print("Type of content:", type(content))
    print("--- File Contents ---")
    print(content)
```
**Output:**
```text
Type of content: <class 'str'>
--- File Contents ---
theme=dark
auto_save=true
font_size=14
```

### 2. `.readline()`
Reads a single line from the file, returning it as a string. Each call to `.readline()` advances the file cursor to the next line. If it reaches the end of the file, it returns an empty string `""`.

**Input:**
```python
with open("settings.txt", "r") as file:
    line1 = file.readline()
    line2 = file.readline()
    print("Line 1:", line1, end="")  # end="" prevents printing double newlines
    print("Line 2:", line2, end="")
```
**Output:**
```text
Line 1: theme=dark
Line 2: auto_save=true
```

### 3. `.readlines()`
Reads all the lines in the file and returns them as a **list of strings**, where each string represents a line including its trailing newline character (`\n`).

**Input:**
```python
with open("settings.txt", "r") as file:
    lines = file.readlines()
    print("Lines List:", lines)
```
**Output:**
```text
Lines List: ['theme=dark\n', 'auto_save=true\n', 'font_size=14\n']
```

---

## 5. Memory-Efficient Line Iteration (Best Practice)

Using `.read()` or `.readlines()` loads the **entire** file into your system's RAM at once. If you are processing a massive database export or a 10 Gigabyte server log file, your program will crash due to running out of memory!

The most Pythonic and memory-efficient way to read a file line-by-line is to loop over the file object directly. This uses **lazy loading**—reading only one line into memory at a time.

**Input:**
```python
with open("settings.txt", "r") as file:
    print("--- Looping Over File Object ---")
    for index, line in enumerate(file, start=1):
        # Use .strip() to clean up trailing newlines and whitespace
        cleaned_line = line.strip()
        print(f"Line {index}: {cleaned_line}")
```
**Output:**
```text
--- Looping Over File Object ---
Line 1: theme=dark
Line 2: auto_save=true
Line 3: font_size=14
```
