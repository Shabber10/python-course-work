# 3. File Cursor and OS Operations

When a file is opened, Python keeps track of the current position inside it using a virtual bookmark called a **File Cursor** (or **File Pointer**). Knowing how to locate and move this cursor gives you granular control over reading and writing.

---

## 1. The File Cursor (Pointer)

- When you open a file in read (`'r'`) or write (`'w'`) mode, the cursor starts at position `0` (the beginning of the file).
- When you open a file in append (`'a'`) mode, the cursor starts at the very end of the file.
- As you read or write characters, the cursor automatically advances forward.

```text
               FILE CURSOR PROGRESSION:
  
  Initial state:               After reading 5 characters:
  ┌───┬───┬───┬───┬───┐        ┌───┬───┬───┬───┬───┐
  │ H │ e │ l │ l │ o │        │ H │ e │ l │ l │ o │
  └───┴───┴───┴───┴───┘        └───┴───┴───┴───┴───┘
  ▲                            ▲
  │ (Cursor at index 0)        │ (Cursor advanced to index 5)
```

---

## 2. Cursor Methods: `.tell()` and `.seek()`

Python provides two methods to query and move the cursor:

### 1. `.tell()`
Returns an integer representing the current position of the file cursor (in bytes from the start of the file).

### 2. `.seek(offset, whence)`
Moves the file cursor to a new position.
- `offset`: The number of bytes to move.
- `whence`: The reference point:
  - `0` (default): Start of the file.
  - `1`: Current position.
  - `2`: End of the file.

> [!NOTE]
> In standard Python text files, you can only seek relative to the beginning of the file (whence = `0`). Seeking relative to the current position (`1`) or end (`2`) is only allowed if the offset is `0` (e.g. `file.seek(0, 2)` to jump to the end).

Let’s see this in action:

**Input:**
```python
# Create a test file
with open("cursor_demo.txt", "w") as file:
    file.write("PythonCode")

# Verify seek/tell behavior
with open("cursor_demo.txt", "r") as file:
    print("1. Cursor start position:", file.tell())
    
    # Read 6 characters
    chunk = file.read(6)
    print(f"2. Read chunk: '{chunk}' | Cursor position now:", file.tell())
    
    # Reset cursor to the beginning (0)
    file.seek(0)
    print("3. Reset cursor to start. Position:", file.tell())
    print("4. Read again from start:", file.read(6))
    
    # Jump to a specific byte (offset 6)
    file.seek(6)
    print("5. Jumped to offset 6. Remaining content:", file.read())
```
**Output:**
```text
1. Cursor start position: 0
2. Read chunk: 'Python' | Cursor position now: 6
3. Reset cursor to start. Position: 0
4. Read again from start: Python
5. Jumped to offset 6. Remaining content: Code
```

## 3. OS File Operations

Reading and writing files is handled by Python's built-in file object. However, if you need to manage the files on the hard drive (such as checking if a file exists, renaming it, or deleting it), you must import Python's built-in **`os` module**.

### A. Checking File Existence
Before trying to open or modify a file, it is best practice to verify it actually exists to prevent your program from crashing.

```python
import os

# Check if file exists
if os.path.exists("settings.txt"):
    print("✅ Settings file found!")
else:
    print("❌ Settings file is missing.")
```

### B. Renaming a File
Use `os.rename(old_name, new_name)` to change a file's name or move it to a new directory.

```python
import os

# Rename file
if os.path.exists("cursor_demo.txt"):
    os.rename("cursor_demo.txt", "navigation_demo.txt")
    print("📂 File renamed successfully.")
```

### C. Deleting a File
Use `os.remove(file_path)` to permanently delete a file.

```python
import os

# Delete file
if os.path.exists("navigation_demo.txt"):
    os.remove("navigation_demo.txt")
    print("🗑️ File deleted from disk.")
```

### D. Creating a Directory (os.mkdir)
Use `os.mkdir(directory_name)` to create a new folder in the specified path.

**Input:**
```python
import os

# Create a folder in the current working directory
if not os.path.exists("myfolder"):
    os.mkdir("myfolder")
    print("📁 Folder 'myfolder' created successfully.")
```
**Output:**
```text
📁 Folder 'myfolder' created successfully.
```

### E. Getting Absolute Paths (os.path.abspath)
Use `os.path.abspath(path)` to resolve relative paths (like `.` or `filename.txt`) into their full, absolute path starting from the root directory.

**Input:**
```python
import os

# Get full absolute path of a file
full_path = os.path.abspath("settings.txt")
# Output will display the full path starting from the drive letter
print("Is absolute?", os.path.isabs(full_path))
```
**Output:**
```text
Is absolute? True
```

### F. Listing Directory Contents (os.listdir)
Use `os.listdir(directory_path)` to retrieve a list of names of all files and directories inside the specified folder.

**Input:**
```python
import os

# List files in the current working directory
current_files = os.listdir(".")
print("Files in workspace (list):", type(current_files))
```
**Output:**
```text
Files in workspace (list): <class 'list'>
```

### G. Formatting File Timestamps (time.ctime)
When querying file metadata (such as modification time `os.path.getmtime`), the operating system returns a raw timestamp (seconds since the Epoch). To convert it to a human-readable string representation, we use the `time.ctime()` function from the built-in `time` module.

**Input:**
```python
import os
import time

# Create a temporary file
with open("time_test.txt", "w") as f:
    f.write("Hello Time")

# Get modification time timestamp
mtime_raw = os.path.getmtime("time_test.txt")
print("Raw Timestamp      :", mtime_raw)

# Convert to human-readable string
readable_time = time.ctime(mtime_raw)
print("Readable Time string:", readable_time)

# Clean up
os.remove("time_test.txt")
```
**Output:**
```text
Raw Timestamp      : 1780963200.0
Readable Time string: Thu Jul  9 10:12:30 2026
```

## 3.5. High-Level File Operations: The `shutil` Module

While the `os` module handles basic tasks like creating folders and deleting files, it cannot copy files or delete non-empty folders. For high-level file and folder operations, Python provides the built-in **`shutil`** (shell utilities) module.

### Common `shutil` Operations:
- `shutil.copy(source, destination)`: Copies a file to a new location.
- `shutil.move(source, destination)`: Moves a file or folder to a new location.
- `shutil.rmtree(folder_path)`: Deletes an entire folder and all of its contents (recursive delete).

**Input:**
```python
import shutil
import os

# 1. Copy a file
# Create a temporary file to copy
with open("settings.txt", "w") as f:
    f.write("theme=dark")

shutil.copy("settings.txt", "settings_backup.txt")
print("📦 Backup file created via shutil.copy().")

# 2. Safely clean up directories and backups
if os.path.exists("myfolder"):
    # Delete folder and all its contents recursively
    shutil.rmtree("myfolder")
    print("🗑️ Non-empty folder 'myfolder' removed via shutil.rmtree().")

# Clean up settings files
os.remove("settings.txt")
os.remove("settings_backup.txt")
```
**Output:**
```text
📦 Backup file created via shutil.copy().
🗑️ Non-empty folder 'myfolder' removed via shutil.rmtree().
```

## 4. Safe File Operations (Best Practice Pattern)

Below is the standard, production-ready pattern for safely cleaning up or renaming files.

**Input:**
```python
import os

def delete_temp_file(filepath):
    # Verify file exists before removing to avoid FileNotFoundError
    if os.path.exists(filepath):
        try:
            os.remove(filepath)
            print(f"✅ Successfully deleted temporary file: {filepath}")
        except PermissionError:
            print(f"❌ Cannot delete {filepath}. File is currently locked or in use.")
    else:
        print(f"⚠️ Deletion skipped. File {filepath} does not exist.")

# Test the safe cleanup utility
delete_temp_file("non_existent_temp.txt")
```
**Output:**
```text
⚠️ Deletion skipped. File non_existent_temp.txt does not exist.
```
