# Match-Case Statement (Structural Pattern Matching)

Introduced in **Python 3.10**, the `match-case` statement provides a clean, readable, and powerful way to handle multiple conditional branches. It is Python's equivalent to the `switch-case` statement found in languages like C, Java, or JavaScript, but with much more power.

It allows you to compare a variable (or structure) against a series of patterns and execute code based on the first matching pattern.

## Syntax
```python
match subject_variable:
    case pattern1:
        # Code block
    case pattern2:
        # Code block
    case _:
        # Default block (optional, acts like 'else')
```
*Note: The `_` (underscore) acts as a wildcard that matches anything. It's used as the default case.*

---

## Example 1: Basic Usage
Replacing a long `if-elif-else` chain.

```python
def get_day_type(day):
    match day.lower():
        case "saturday" | "sunday":   # The '|' symbol means OR
            return "Weekend"
        case "monday" | "tuesday" | "wednesday" | "thursday" | "friday":
            return "Weekday"
        case _:
            return "Invalid day!"

print(get_day_type("Sunday"))   # Output: Weekend
print(get_day_type("Monday"))   # Output: Weekday
print(get_day_type("Funday"))   # Output: Invalid day!
```

**Output:**
```text
Weekend
Weekday
Invalid day!
```

## Example 2: Matching Data Structures (Lists/Tuples)
The real power of `match-case` lies in its ability to unpack and match complex data structures.

```python
def process_command(command):
    match command.split():
        case ["quit"]:
            print("Quitting the program...")
        case ["load", filename]:
            print(f"Loading file: {filename}")
        case ["save", filename, format]:
            print(f"Saving file {filename} as {format}")
        case _:
            print("Command not recognized.")

process_command("quit")                 # Output: Quitting the program...
process_command("load data.txt")        # Output: Loading file: data.txt
process_command("save config.json json")# Output: Saving file config.json as json
process_command("dance")                # Output: Command not recognized.
```

**Output:**
```text
Quitting the program...
Loading file: data.txt
Saving file config.json as json
Command not recognized.
```


---

## Example 3: Adding Conditions (Guards)
You can add an `if` condition to a `case` to make the match even more specific. This is called a **guard**.

```python
def check_temperature(temp):
    match temp:
        case t if t < 0:
            print("Freezing!")
        case t if 0 <= t <= 25:
            print("Pleasant weather.")
        case t if t > 25:
            print("It's hot outside.")
        case _:
            print("Invalid input")

check_temperature(-5)  # Output: Freezing!
check_temperature(20)  # Output: Pleasant weather.
```

**Output:**
```text
Freezing!
Pleasant weather.
```


## Why use `match-case`?
1. **Readability**: It makes handling multiple complex conditions much cleaner than chained `elif` blocks.
2. **Data Unpacking**: It elegantly extracts data from lists, tuples, and dictionaries directly inside the conditional structure.
