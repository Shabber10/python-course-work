# 3. The Itertools Module

Python's built-in **`itertools`** module is a collection of tools for handling iterators. It provides fast, memory-efficient functions that can be combined to perform complex iterator algebra.

Two of the most popular combinatoric tools in this module are **`combinations`** and **`permutations`**.

---

## 1. Permutations: `itertools.permutations()`
A **permutation** is an arrangement of elements where the **order of elements matters**. 
For example, `('a', 'b')` and `('b', 'a')` are two different permutations of the set `{'a', 'b'}`.

### Syntax:
```python
from itertools import permutations
permutations(iterable, r=None)
```
- `iterable`: The sequence from which to make permutations.
- `r`: The length of each permutation sequence. If omitted or `None`, it defaults to the length of the iterable (generating all possible full-length arrangements).

**Input:**
```python
from itertools import permutations

# Get all permutations of length 2 from 'abc'
perms = permutations('abc', 2)

# Convert iterator to list of tuples
perms_list = list(perms)
print("Permutations (raw tuples):", perms_list)

# Join the tuples back into strings for clean output
perms_strings = [''.join(p) for p in perms_list]
print("Permutations (joined strings):", perms_strings)
```
**Output:**
```text
Permutations (raw tuples): [('a', 'b'), ('a', 'c'), ('b', 'a'), ('b', 'c'), ('c', 'a'), ('c', 'b')]
Permutations (joined strings): ['ab', 'ac', 'ba', 'bc', 'ca', 'cb']
```

---

## 2. Combinations: `itertools.combinations()`
A **combination** is a selection of elements where the **order of elements does NOT matter**.
For example, `('a', 'b')` and `('b', 'a')` represent the same combination. Therefore, only one of them is returned (sorted according to the input order).

### Syntax:
```python
from itertools import combinations
combinations(iterable, r)
```
- `r`: The length of each combination sequence. Unlike permutations, $r$ is a **required** argument.

**Input:**
```python
from itertools import combinations

# Get all combinations of length 2 from 'abc'
combs = combinations('abc', 2)

# Convert iterator to list of tuples
combs_list = list(combs)
print("Combinations (raw tuples):", combs_list)

# Join the tuples back into strings for clean output
combs_strings = [''.join(c) for c in combs_list]
print("Combinations (joined strings):", combs_strings)
```
**Output:**
```text
Combinations (raw tuples): [('a', 'b'), ('a', 'c'), ('b', 'c')]
Combinations (joined strings): ['ab', 'ac', 'bc']
```

---

## 3. Comparison summary
Let's compare the results of permutations vs. combinations for the string `'abc'` with length `2`:

| Input | Function | Length `r` | Output Count | Unique Output List |
| :---: | :---: | :---: | :---: | :---: |
| `'abc'` | `combinations` | `2` | **3** | `['ab', 'ac', 'bc']` |
| `'abc'` | `permutations` | `2` | **6** | `['ab', 'ac', 'ba', 'bc', 'ca', 'cb']` |

> [!TIP]
> Both `combinations` and `permutations` are evaluated lazily. They return an **iterator** object and do not compute the entire list of arrangements in memory, making them highly efficient even for larger input sequences.
