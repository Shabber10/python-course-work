# 2. Creating and Using Packages

As your codebase grows larger, keeping all code in a single file or directory becomes chaotic. Python uses **modules** and **packages** to manage namespaces and organize files.

- **Module:** A single Python file (`.py` file) containing variables, functions, and classes.
- **Package:** A directory containing multiple modules and a special file named `__init__.py`.

---

## 1. Package Directory Structure
To construct a package named `my_package`, we organize files in a folder structure:

```text
my_project/
│
├── main.py
└── my_package/
    ├── __init__.py
    ├── calculator.py
    └── string_helper.py
```

- **`my_package/`**: The folder representing the package.
- **`__init__.py`**: Tells Python that this directory should be treated as a package. It can be empty or contain initialization code.
- **`calculator.py` and `string_helper.py`**: Modules containing reusable code.
- **`main.py`**: The main entry point script located outside the package directory.

---

## 2. Coding the Modules
Let's see what the individual files in our package look like.

### `my_package/calculator.py`
```python
def add(a, b):
    return a + b

def subtract(a, b):
    return a - b
```

### `my_package/string_helper.py`
```python
def capitalize_word(word):
    return word.capitalize()
```

### `my_package/__init__.py`
This file is executed when the package is imported. We can leave it completely empty, or we can use it to control what modules are exposed by default:
```python
# Optional init code
print("Initializing my_package...")

# Expose specific module functions at package level
from .calculator import add, subtract
from .string_helper import capitalize_word
```

---

## 3. Absolute vs. Relative Imports
When importing modules inside a package, Python supports two syntax types:

### A. Absolute Imports
Specifies the complete path from the project's root folder down to the module.
```python
from my_package.calculator import add
```
- **Pros:** Clear, easy to read, and works regardless of where the calling script is relocated.
- **Cons:** If you rename the package folder, you have to change all imports in all files.

### B. Relative Imports
Specifies the path relative to the current module's directory location using leading dots (`.`):
- A single dot `.` represents the **current directory**.
- A double dot `..` represents the **parent directory**.

```python
# Inside my_package/__init__.py:
from .calculator import add  # Imports 'add' from calculator.py in the same folder
```
- **Pros:** Flexible. You can rename or move the package folder and the internal package imports will not break.
- **Cons:** Only works for modules inside a package; running a file with relative imports directly (as the main script) raises a `ImportError`.

---

## 4. Importing from a Package
Once your package structure is set up, you can import and use its contents in `main.py` in multiple ways:

### Method A: Importing Specific Modules
**Input:**
```python
from my_package import calculator
from my_package import string_helper

print(calculator.add(5, 3))
print(string_helper.capitalize_word("hello"))
```
**Output:**
```text
8
Hello
```

### Method B: Importing Specific Functions Directly
**Input:**
```python
from my_package.calculator import add
from my_package.string_helper import capitalize_word

print(add(10, 20))
print(capitalize_word("python"))
```
**Output:**
```text
30
Python
```

### Method C: Exposing at Package Level via `__init__.py`
Because we imported `add` and `capitalize_word` inside `__init__.py`, we can import them directly from `my_package`:

**Input:**
```python
import my_package

print(my_package.add(15, 10))
print(my_package.capitalize_word("world"))
```
**Output:**
```text
Initializing my_package...
25
World
```
