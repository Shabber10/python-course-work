# 1. Iterators and Custom Iterators

In Python, iteration is a core concept that powers loops. Under the hood, iteration relies on the **iterator protocol**.

---

## 1. Iterables vs. Iterators
It is crucial to understand the difference between these two related terms:

1. **Iterable:** An object that can return its members one at a time. It implements the `__iter__()` method (or has a `__getitem__()` method). Examples: Lists, Tuples, Strings, Dictionaries, Sets.
2. **Iterator:** An object that represents a stream of data. It implements the iterator protocol with:
   - `__iter__()`: Returns the iterator object itself.
   - `__next__()`: Returns the next item in the stream. If no elements are left, it raises `StopIteration`.

> [!NOTE]
> Every **Iterator** is also an **Iterable** (since it has `__iter__()`), but not every **Iterable** is an **Iterator**. For example, a list is an iterable, but it is *not* an iterator (you cannot call `next(list)` directly without first converting it).

**Input:**
```python
my_list = [10, 20]

# Try to call next() on the list directly -> Raises TypeError!
try:
    next(my_list)
except TypeError as e:
    print("Error:", e)

# Convert list (iterable) to an iterator using iter()
my_iterator = iter(my_list)
print("Iterator object:", my_iterator)

# Now we can retrieve values using next()
print(next(my_iterator))
print(next(my_iterator))
```
**Output:**
```text
Error: 'list' object is not an iterator
Iterator object: <list_iterator object at 0x...>
10
20
```

---

## 2. How the `for` Loop Works Under the Hood
When you write a `for` loop:
```python
for item in iterable:
    # do something
```
Python converts the loop internally into the following:
```python
# Create an iterator object from the iterable
iterator_obj = iter(iterable)

# Infinite loop
while True:
    try:
        # Get the next item
        item = next(iterator_obj)
        # Execute loop body
    except StopIteration:
        # If StopIteration is raised, break from loop
        break
```

---

## 3. The `StopIteration` Exception
Unlike standard errors (like `ValueError` or `TypeError`) which indicate that something went wrong in your code, `StopIteration` is a **control-flow exception**. 

- It does not represent a bug.
- It is raised by an iterator's `__next__()` method to signal to the calling loop that the sequence is complete and there are no further values to read.

---

## 4. Creating a Custom Iterator Class
To make a class act as an iterator, we implement both `__iter__()` and `__next__()`.

Here is a custom iterator class `Counter` that counts from a `start` value up to a `limit`.

**Input:**
```python
class Counter:
    def __init__(self, start, limit):
        self.current = start
        self.limit = limit

    # The __iter__ method must return the iterator object itself
    def __iter__(self):
        return self

    # The __next__ method must return the next value, and raise StopIteration when done
    def __next__(self):
        if self.current <= self.limit:
            val = self.current
            self.current += 1
            return val
        else:
            raise StopIteration

# Using our custom iterator in a for loop!
c = Counter(1, 4)
for num in c:
    print(num)
```
**Output:**
```text
1
2
3
4
```
