# 2. Instance, Class, and Static Methods

When writing classes in Python, it's essential to understand where data is stored (Class vs. Instance variables) and what scope your methods can access. Python supports three types of methods: **Instance**, **Class**, and **Static** methods.

---

## 1. Class Variables vs. Instance Variables

Let's set up a gaming server where players join a lobby.

- **Instance Variables (Specific to the object):** Defined inside the constructor (`__init__`) using `self.`. Each player has their own copy of `username`, `hp`, and `ammo`.
- **Class Variables (Shared by all objects):** Defined directly in the class body. Every player shares the exact same `server_name` and `max_players` limit.

Let’s see the difference in code:

**Input:**
```python
class GameLobby:
    # Class variables (shared by all instances)
    server_name = "Asia-South"
    max_players = 50

    def __init__(self, username, hp=100):
        # Instance variables (unique to each instance)
        self.username = username
        self.hp = hp
        self.ammo = 30

# Create player instances
p1 = GameLobby("ApexPredator")
p2 = GameLobby("ShadowNinja")

# 1. Read instance variables
print(f"p1 username: {p1.username}, hp: {p1.hp}, ammo: {p1.ammo}")
print(f"p2 username: {p2.username}, hp: {p2.hp}, ammo: {p2.ammo}")

# 2. Read class variables (accessed via instances or the class name)
print("p1 Server:", p1.server_name)
print("p2 Server:", p2.server_name)
print("Class Server:", GameLobby.server_name)
```
**Output:**
```text
p1 username: ApexPredator, hp: 100, ammo: 30
p2 username: ShadowNinja, hp: 100, ammo: 30
p1 Server: Asia-South
p2 Server: Asia-South
Class Server: Asia-South
```

## 2. The Three Types of Methods

Python methods can be configured to access either the **instance state**, the **class state**, or **no state at all**.

```text
  ┌────────────────────────────────────────────────────────┐
  │                 TYPES OF PYTHON METHODS                │
  └────────────────────────────────────────────────────────┘
       │                        │                        │
  ┌────┴─────────────┐    ┌─────┴────────────┐    ┌──────┴───────────┐
  │ Instance Method  │    │   Class Method   │    │  Static Method   │
  ├──────────────────┤    ├──────────────────┤    ├──────────────────┤
  │ - Receives 'self'│    │ - Receives 'cls' │    │ - Receives no    │
  │ - Accesses/edits │    │ - Accesses/edits │    │   implied arg    │
  │   instance state │    │   class-level    │    │ - Utility helper │
  │   and class state│    │   variables      │    │   functions      │
  └──────────────────┘    └──────────────────┘    └──────────────────┘
```

---

## 3. Instance Methods
Instance methods are the default methods in Python. They automatically receive the calling object as the first parameter, conventionally named `self`.
- **Use Case:** When the action needs to read or modify a specific object's data (e.g., firing a weapon, which decreases only *that* player's ammo).

**Input:**
```python
class Player:
    def __init__(self, username, ammo=30):
        self.username = username
        self.ammo = ammo

    # Instance method
    def fire_weapon(self, shots):
        if self.ammo >= shots:
            self.ammo -= shots
            print(f"🔫 {self.username} fired {shots} shots! Ammo left: {self.ammo}")
        else:
            print(f"⚠️ Out of ammo! {self.username} has {self.ammo} bullets left.")

p = Player("GhostRider", ammo=10)
p.fire_weapon(3)
p.fire_weapon(10)  # Fails due to low ammo
```
**Output:**
```text
🔫 GhostRider fired 3 shots! Ammo left: 7
⚠️ Out of ammo! GhostRider has 7 bullets left.
```

---

## 4. Class Methods (`@classmethod`)
Class methods are bound to the class, not the instance. They automatically receive the class itself as the first parameter, conventionally named `cls`. 
- **Setup:** Decorated with `@classmethod`.
- **Use Case:** To access or modify class-level variables (e.g. changing the game lobby server name for all active players), or to build **factory methods** that create new instances from alternative structures.

**Input:**
```python
class GameLobby:
    server_name = "Asia-South"
    active_players = 0

    def __init__(self, username):
        self.username = username
        # Increment active player counter on instantiation
        GameLobby.active_players += 1

    # Class method to modify class variables
    @classmethod
    def change_server(cls, new_server):
        cls.server_name = new_server
        print(f"🌐 Server migrated to: {cls.server_name}")

    # Class method as a Factory (creates a template Bot player)
    @classmethod
    def create_bot(cls, bot_id):
        # returns an instance of this class (GameLobby)
        return cls(f"BOT_{bot_id}")

# 1. Change server using class method
GameLobby.change_server("Europe-Central")

# 2. Check server name on a new player
p = GameLobby("Viper")
print(f"Player {p.username} is connected to: {p.server_name}")

# 3. Create a player using the factory class method
bot = GameLobby.create_bot(99)
print(f"Spawned: {bot.username}")
print(f"Total Active Players: {GameLobby.active_players}")
```
**Output:**
```text
🌐 Server migrated to: Europe-Central
Player Viper is connected to: Europe-Central
Spawned: BOT_99
Total Active Players: 2
```

---

## 5. Static Methods (`@staticmethod`)
Static methods behave exactly like standard, plain functions, but they live inside the namespace of a class. They do **not** receive `self` or `cls` parameters, meaning they cannot access or modify instance or class variables.
- **Setup:** Decorated with `@staticmethod`.
- **Use Case:** When an action does not depend on any player or server data but is conceptually related to the class namespace (e.g., verifying if a connection ping is high/poor).

**Input:**
```python
class NetworkTool:
    # Static method
    @staticmethod
    def evaluate_ping(latency_ms):
        if latency_ms < 50:
            return "🟢 Excellent"
        elif latency_ms < 150:
            return "🟡 Average"
        else:
            return "🔴 Bad (Lagging)"

# We can call static methods directly via class name without instantiating objects!
ping_status = NetworkTool.evaluate_ping(185)
print("Current connection rating:", ping_status)
```
**Output:**
```text
Current connection rating: 🔴 Bad (Lagging)
```

---

## 6. Summary Comparison Table

| Method Type | Decorator | First Parameter | Can Access Instance State (`self`)? | Can Access Class State (`cls`)? | Typical Use Case |
| :--- | :--- | :--- | :---: | :---: | :--- |
| **Instance Method** | None | `self` | **Yes** | **Yes** | Modify individual object attributes (e.g., player health, position). |
| **Class Method** | `@classmethod` | `cls` | **No** | **Yes** | Modify class-wide variables, or build custom factory initializers. |
| **Static Method** | `@staticmethod` | None | **No** | **No** | Utility functions related to the class topic (e.g., calculations/validations). |
