# 2. Instance, Class, and Static Methods

When writing classes in Python, it's essential to understand where data is stored (Class vs. Instance variables) and what scope your methods can access. Python supports three types of methods: **Instance**, **Class**, and **Static** methods.

---

## 1. Class Variables vs. Instance Variables

Let's set up a system to track channels on **YouTube**.

- **Instance Variables (Specific to the object):** Defined inside the constructor (`__init__`) using `self.`. Each channel has its own unique `channel_name` and `subscribers` count.
- **Class Variables (Shared by all objects):** Defined directly in the class body. Every channel shares the same `platform_name` and `video_resolution_limit`.

Let’s see the difference in code:

**Input:**
```python
class YouTubeChannel:
    # Class variables (shared by all instances)
    platform_name = "YouTube"
    video_resolution_limit = "4K"

    def __init__(self, channel_name, subscribers=0):
        # Instance variables (unique to each instance)
        self.channel_name = channel_name
        self.subscribers = subscribers

# Create channel instances
ch1 = YouTubeChannel("CodeBasics", 12000)
ch2 = YouTubeChannel("TravelVlogs", 8500)

# 1. Read instance variables
print(f"ch1 channel name: {ch1.channel_name}, Subscribers: {ch1.subscribers}")
print(f"ch2 channel name: {ch2.channel_name}, Subscribers: {ch2.subscribers}")

# 2. Read class variables (accessed via instances or the class name)
print("ch1 Platform:", ch1.platform_name)
print("ch2 Platform:", ch2.platform_name)
print("Class Platform Name:", YouTubeChannel.platform_name)
```
**Output:**
```text
ch1 channel name: CodeBasics, Subscribers: 12000
ch2 channel name: TravelVlogs, Subscribers: 8500
ch1 Platform: YouTube
ch2 Platform: YouTube
Class Platform Name: YouTube
```

## 2. The Three Types of Methods

Python methods can be configured to access either the **instance state**, the **class state**, or **no state at all**.

```text
  ┌────────────────────────────────────────────────────────┐
  │                 TYPES OF PYTHON METHODS                │
  └────────────────────────────────────────────────────────┘
       │                        │                        │
  ┌────┴─────────────┐    ┌─────┴────────────┐    ┌──────┴───────────┐
  │ Instance Method  │    │   Class Method   │    │  Static Method   │
  ├──────────────────┤    ├──────────────────┤    ├──────────────────┤
  │ - Receives 'self'│    │ - Receives 'cls' │    │ - Receives no    │
  │ - Accesses/edits │    │ - Accesses/edits │    │   implied arg    │
  │   instance state │    │   class-level    │    │ - Utility helper │
  │   and class state│    │   variables      │    │   functions      │
  └──────────────────┘    └──────────────────┘    └──────────────────┘
```

---

## 3. Instance Methods
Instance methods are the default methods in Python. They automatically receive the calling object as the first parameter, conventionally named `self`.
- **Use Case:** When the action needs to read or modify a specific object's data (e.g., liking a specific post, which increments only *that* post's likes).

**Input:**
```python
class InstagramPost:
    def __init__(self, author, caption):
        self.author = author
        self.caption = caption
        self.likes = 0

    # Instance method: modifies the likes of this specific post instance
    def like(self):
        self.likes += 1
        print(f"❤️ Post by {self.author} liked! Total Likes: {self.likes}")

post1 = InstagramPost("john", "Had a great weekend!")
post1.like()
post1.like()
```
**Output:**
```text
❤️ Post by john liked! Total Likes: 1
❤️ Post by john liked! Total Likes: 2
```

---

## 4. Class Methods (`@classmethod`)
Class methods are bound to the class, not the instance. They automatically receive the class itself as the first parameter, conventionally named `cls`. 
- **Setup:** Decorated with `@classmethod`.
- **Use Case:** To access or modify class-level variables (e.g., keeping track of the total number of bank accounts created), or to build **factory methods** that construct objects with preset behaviors.

**Input:**
```python
class BankAccount:
    total_accounts_created = 0

    def __init__(self, owner, account_type, balance=0):
        self.owner = owner
        self.account_type = account_type
        self.balance = balance
        # Increment class-level account counter on instantiation
        BankAccount.total_accounts_created += 1

    # Class method to get total accounts created
    @classmethod
    def get_total_accounts(cls):
        print(f"🏦 Total Bank Accounts Registered: {cls.total_accounts_created}")

    # Class method as a Factory (creates a Savings account automatically)
    @classmethod
    def create_savings(cls, owner, initial_deposit):
        # returns a new instance of this class (BankAccount)
        return cls(owner, "Savings", initial_deposit)

# 1. Create a standard account
acct1 = BankAccount("emma", "Checking", 500)

# 2. Spawn a savings account using the factory class method
acct2 = BankAccount.create_savings("sam", 1000)
print(f"Account Owner: {acct2.owner}, Type: {acct2.account_type}, Balance: {acct2.balance}")

# 3. Read total count using class method
BankAccount.get_total_accounts()
```
**Output:**
```text
Account Owner: sam, Type: Savings, Balance: 1000
🏦 Total Bank Accounts Registered: 2
```

## 5. Static Methods (`@staticmethod`)
Static methods behave exactly like standard, plain functions, but they live inside the namespace of a class. They do **not** receive `self` or `cls` parameters, meaning they cannot access or modify instance or class variables.
- **Setup:** Decorated with `@staticmethod`.
- **Use Case:** When an action does not depend on any specific object or class-wide variables, but is conceptually linked to the class namespace (e.g., verifying if a password is secure).

**Input:**
```python
class PasswordTool:
    # Static method: checks if password is secure (independent utility helper)
    @staticmethod
    def is_strong(password):
        # Must be at least 8 characters long
        return len(password) >= 8

# We call the static method directly via the class name (no instantiation needed!)
print("Is '1234' secure?", PasswordTool.is_strong("1234"))
print("Is 'safePassword123' secure?", PasswordTool.is_strong("safePassword123"))
```
**Output:**
```text
Is '1234' secure? False
Is 'safePassword123' secure? True
```

## 6. Comprehensive Practical Example: Redbus Seat Booking

To see how Class Variables, Class Methods, and Instance Methods work together in a single system, let's analyze a bus seat booking system like **Redbus**.

In this system:
- **Class Variable (`seats`):** A shared dictionary representing all 10 seats (even-numbered seats are pre-booked, odd-numbered seats are available).
- **Class Method (`displayseats`):** Accesses and displays the class-level seat database.
- **Instance Method (`booking`):** An action executed by individual users to claim a specific seat.

**Input:**
```python
class Redbus:
    # Class variable: Shared dictionary representing seats
    seats = {}
    for i in range(1, 11):
        if i % 2 == 0:
            seats[i] = 'Booked'
        else:
            seats[i] = 'Available'

    # Class method to display the seating chart of the bus
    @classmethod
    def displayseats(cls):
        print("--- Bus Seat Status ---")
        for seat, status in cls.seats.items():
            print(f'| Seat {seat} | {status} |')
        print("-----------------------")

    # Instance method for a user to book a seat
    def booking(self, seatno):
        if Redbus.seats[seatno] == 'Booked':
            print(f'❌ Seat {seatno} is already Booked!')
        else:
            Redbus.seats[seatno] = 'Booked'  # Update the shared database!
            print(f'✅ Seat {seatno} has been Successfully Booked!')

# Create passenger instances
passenger = Redbus()

# Show initial seating layout
passenger.displayseats()

# Passengers book seats
passenger.booking(1)
passenger.booking(7)
passenger.booking(9)

# Try booking an already booked seat
passenger.booking(1)

# Show updated seating layout
passenger.displayseats()
```
**Output:**
```text
--- Bus Seat Status ---
| Seat 1 | Available |
| Seat 2 | Booked |
| Seat 3 | Available |
| Seat 4 | Booked |
| Seat 5 | Available |
| Seat 6 | Booked |
| Seat 7 | Available |
| Seat 8 | Booked |
| Seat 9 | Available |
| Seat 10 | Booked |
-----------------------
✅ Seat 1 has been Successfully Booked!
✅ Seat 7 has been Successfully Booked!
✅ Seat 9 has been Successfully Booked!
❌ Seat 1 is already Booked!
--- Bus Seat Status ---
| Seat 1 | Booked |
| Seat 2 | Booked |
| Seat 3 | Available |
| Seat 4 | Booked |
| Seat 5 | Available |
| Seat 6 | Booked |
| Seat 7 | Booked |
| Seat 8 | Booked |
| Seat 9 | Booked |
| Seat 10 | Booked |
-----------------------
```
> [!NOTE]
> In programming, be careful not to confuse comparison (`==`) with assignment (`=`). When booking a seat, we use `Redbus.seats[seatno] = 'Booked'` to modify the database. If we accidentally wrote `==`, Python would simply compare the values and do nothing, leaving the seat unreserved!

## 7. Summary Comparison Table

| Method Type | Decorator | First Parameter | Can Access Instance State (`self`)? | Can Access Class State (`cls`)? | Typical Use Case |
| :--- | :--- | :--- | :---: | :---: | :--- |
| **Instance Method** | None | `self` | **Yes** | **Yes** | Modify individual object attributes (e.g., post likes, user balance). |
| **Class Method** | `@classmethod` | `cls` | **No** | **Yes** | Modify class-wide variables, or build custom factory initializers. |
| **Static Method** | `@staticmethod` | None | **No** | **No** | Utility functions related to the class topic (e.g., checking inputs/formats). |
