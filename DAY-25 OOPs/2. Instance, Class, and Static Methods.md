# 2. Instance, Class, and Static Methods

When writing classes in Python, it's essential to understand where data is stored (Class vs. Instance variables) and what scope your methods can access. Python supports three types of methods: **Instance**, **Class**, and **Static** methods.

---

## 1. Class Variables vs. Instance Variables

Let's set up a match in **Free Fire**.

- **Instance Variables (Specific to the character):** Defined inside the constructor (`__init__`) using `self.`. Each character has their own unique `name`, `hp`, and active `weapon`.
- **Class Variables (Shared by all characters):** Defined directly in the class body. Every character shares the same `game_title` and `max_hp_limit` settings.

Let’s see the difference in code:

**Input:**
```python
class FreeFireCharacter:
    # Class variables (shared by all instances)
    game_title = "Free Fire"
    max_hp_limit = 200

    def __init__(self, name, hp=200):
        # Instance variables (unique to each instance)
        self.name = name
        self.hp = hp
        self.ammo = 30

# Create character instances
c1 = FreeFireCharacter("Alok")
c2 = FreeFireCharacter("Kelly", hp=180)

# 1. Read instance variables
print(f"c1 character name: {c1.name}, HP: {c1.hp}, ammo: {c1.ammo}")
print(f"c2 character name: {c2.name}, HP: {c2.hp}, ammo: {c2.ammo}")

# 2. Read class variables (accessed via instances or the class name)
print("c1 Game:", c1.game_title)
print("c2 Game:", c2.game_title)
print("Class Game Title:", FreeFireCharacter.game_title)
```
**Output:**
```text
c1 character name: Alok, HP: 200, ammo: 30
c2 character name: Kelly, HP: 180, ammo: 30
c1 Game: Free Fire
c2 Game: Free Fire
Class Game Title: Free Fire
```

## 2. The Three Types of Methods

Python methods can be configured to access either the **instance state**, the **class state**, or **no state at all**.

```text
  ┌────────────────────────────────────────────────────────┐
  │                 TYPES OF PYTHON METHODS                │
  └────────────────────────────────────────────────────────┘
       │                        │                        │
  ┌────┴─────────────┐    ┌─────┴────────────┐    ┌──────┴───────────┐
  │ Instance Method  │    │   Class Method   │    │  Static Method   │
  ├──────────────────┤    ├──────────────────┤    ├──────────────────┤
  │ - Receives 'self'│    │ - Receives 'cls' │    │ - Receives no    │
  │ - Accesses/edits │    │ - Accesses/edits │    │   implied arg    │
  │   instance state │    │   class-level    │    │ - Utility helper │
  │   and class state│    │   variables      │    │   functions      │
  └──────────────────┘    └──────────────────┘    └──────────────────┘
```

---

## 3. Instance Methods
Instance methods are the default methods in Python. They automatically receive the calling object as the first parameter, conventionally named `self`.
- **Use Case:** When the action needs to read or modify a specific object's data (e.g., firing a weapon, which decreases only *that* weapon's current ammo clip).

**Input:**
```python
class FreeFireWeapon:
    def __init__(self, name, ammo=30):
        self.name = name
        self.ammo = ammo

    # Instance method: modifies the ammo of this specific gun instance
    def fire(self, shots):
        if self.ammo >= shots:
            self.ammo -= shots
            print(f"🔫 {self.name} fired {shots} shots! Ammo remaining in clip: {self.ammo}")
        else:
            print(f"⚠️ Cannot fire! {self.name} has only {self.ammo} bullets left. Reload!")

weapon = FreeFireWeapon("MP40", ammo=15)
weapon.fire(5)
weapon.fire(12)  # Fails due to low ammo in clip
```
**Output:**
```text
🔫 MP40 fired 5 shots! Ammo remaining in clip: 10
⚠️ Cannot fire! MP40 has only 10 bullets left. Reload!
```

## 4. Class Methods (`@classmethod`)
Class methods are bound to the class, not the instance. They automatically receive the class itself as the first parameter, conventionally named `cls`. 
- **Setup:** Decorated with `@classmethod`.
- **Use Case:** To access or modify class-level variables (e.g., updating the global game mode settings), or to build **factory methods** that construct preset characters.

**Input:**
```python
class FreeFireMatch:
    game_mode = "Squad"
    active_players = 0

    def __init__(self, username):
        self.username = username
        # Increment active player counter on instantiation
        FreeFireMatch.active_players += 1

    # Class method to modify class variables
    @classmethod
    def change_game_mode(cls, new_mode):
        cls.game_mode = new_mode
        print(f"🌐 Game mode updated to: {cls.game_mode}")

    # Class method as a Factory (creates a preset Bot character)
    @classmethod
    def spawn_bot(cls, bot_id):
        # returns a new instance of this class (FreeFireMatch)
        return cls(f"BOT_{bot_id}")

# 1. Change match type using class method
FreeFireMatch.change_game_mode("Clash Squad")

# 2. Check game mode on a newly joined player
p = FreeFireMatch("Viper")
print(f"Player {p.username} joined. Mode: {p.game_mode}")

# 3. Spawn a bot using the factory class method
bot = FreeFireMatch.spawn_bot(77)
print(f"Spawned player: {bot.username}")
print(f"Total Players in Lobby: {FreeFireMatch.active_players}")
```
**Output:**
```text
🌐 Game mode updated to: Clash Squad
Player Viper joined. Mode: Clash Squad
Spawned player: BOT_77
Total Players in Lobby: 2
```

## 5. Static Methods (`@staticmethod`)
Static methods behave exactly like standard, plain functions, but they live inside the namespace of a class. They do **not** receive `self` or `cls` parameters, meaning they cannot access or modify instance or class variables.
- **Setup:** Decorated with `@staticmethod`.
- **Use Case:** When an action does not depend on any specific character's state or class-wide settings, but is conceptually linked to combat actions (e.g., calculating armor damage reduction).

**Input:**
```python
class CombatCalculator:
    # Static method: calculates damage after vest/armor reduction
    @staticmethod
    def calculate_damage(base_damage, armor_level):
        # Level 1 armor: 10% reduction
        # Level 2 armor: 20% reduction
        # Level 3 armor: 30% reduction
        if armor_level == 1:
            return int(base_damage * 0.9)
        elif armor_level == 2:
            return int(base_damage * 0.8)
        elif armor_level == 3:
            return int(base_damage * 0.7)
        return base_damage

# We call the static method directly via the class name (no instantiation needed!)
damage_taken = CombatCalculator.calculate_damage(120, armor_level=3)
print(f"AWM body shot (120 base) against Level 3 Vest deals: {damage_taken} HP damage.")
```
**Output:**
```text
AWM body shot (120 base) against Level 3 Vest deals: 84 HP damage.
```

## 6. Comprehensive Practical Example: Redbus Seat Booking

To see how Class Variables, Class Methods, and Instance Methods work together in a single system, let's analyze a bus seat booking system like **Redbus**.

In this system:
- **Class Variable (`seats`):** A shared dictionary representing all 10 seats (even-numbered seats are pre-booked, odd-numbered seats are available).
- **Class Method (`displayseats`):** Accesses and displays the class-level seat database.
- **Instance Method (`booking`):** An action executed by individual users to claim a specific seat.

**Input:**
```python
class Redbus:
    # Class variable: Shared dictionary representing seats
    seats = {}
    for i in range(1, 11):
        if i % 2 == 0:
            seats[i] = 'Booked'
        else:
            seats[i] = 'Available'

    # Class method to display the seating chart of the bus
    @classmethod
    def displayseats(cls):
        print("--- Bus Seat Status ---")
        for seat, status in cls.seats.items():
            print(f'| Seat {seat} | {status} |')
        print("-----------------------")

    # Instance method for a user to book a seat
    def booking(self, seatno):
        if Redbus.seats[seatno] == 'Booked':
            print(f'❌ Seat {seatno} is already Booked!')
        else:
            Redbus.seats[seatno] = 'Booked'  # Update the shared database!
            print(f'✅ Seat {seatno} has been Successfully Booked!')

# Create passenger instances
kavya = Redbus()

# Show initial seating layout
kavya.displayseats()

# Passengers book seats
kavya.booking(1)
kavya.booking(7)
kavya.booking(9)

# Try booking an already booked seat
kavya.booking(1)

# Show updated seating layout
kavya.displayseats()
```
**Output:**
```text
--- Bus Seat Status ---
| Seat 1 | Available |
| Seat 2 | Booked |
| Seat 3 | Available |
| Seat 4 | Booked |
| Seat 5 | Available |
| Seat 6 | Booked |
| Seat 7 | Available |
| Seat 8 | Booked |
| Seat 9 | Available |
| Seat 10 | Booked |
-----------------------
✅ Seat 1 has been Successfully Booked!
✅ Seat 7 has been Successfully Booked!
✅ Seat 9 has been Successfully Booked!
❌ Seat 1 is already Booked!
--- Bus Seat Status ---
| Seat 1 | Booked |
| Seat 2 | Booked |
| Seat 3 | Available |
| Seat 4 | Booked |
| Seat 5 | Available |
| Seat 6 | Booked |
| Seat 7 | Booked |
| Seat 8 | Booked |
| Seat 9 | Booked |
| Seat 10 | Booked |
-----------------------
```
> [!NOTE]
> In programming, be careful not to confuse comparison (`==`) with assignment (`=`). When booking a seat, we use `Redbus.seats[seatno] = 'Booked'` to modify the database. If we accidentally wrote `==`, Python would simply compare the values and do nothing, leaving the seat unreserved!

## 7. Summary Comparison Table

| Method Type | Decorator | First Parameter | Can Access Instance State (`self`)? | Can Access Class State (`cls`)? | Typical Use Case |
| :--- | :--- | :--- | :---: | :---: | :--- |
| **Instance Method** | None | `self` | **Yes** | **Yes** | Modify individual object attributes (e.g., weapon ammo, character HP). |
| **Class Method** | `@classmethod` | `cls` | **No** | **Yes** | Modify class-wide variables, or build custom factory initializers. |
| **Static Method** | `@staticmethod` | None | **No** | **No** | Utility functions related to the class topic (e.g., combat math, physics checks). |
