# 3. Object Representation and Dunder Methods

Python objects have several built-in hidden behaviors. These behaviors are controlled by special methods called **Dunder Methods** (short for **D**ouble **Under**score), also known as **Magic Methods**. 

These methods are prefixed and suffixed by double underscores (e.g., `__init__`, `__str__`). You rarely call them directly; instead, Python triggers them automatically in response to specific operations.

---

## 1. The Default Object Representation
By default, if you try to print a custom object in Python, it outputs a cryptic string showing the class name and its memory location. This is not helpful for players or developers!

**Input:**
```python
class FreeFireWeapon:
    def __init__(self, name, ammo):
        self.name = name
        self.ammo = ammo

weapon = FreeFireWeapon("AWM", 5)
print(weapon)
```
**Output:**
```text
<__main__.FreeFireWeapon object at 0x0000021A45AB3F40>
```

To customize how our objects are displayed to users and developers, we override the `__str__` and `__repr__` dunder methods.

---

## 2. The `__str__` Method
The `__str__` method returns an informal, user-friendly string representation of the object. It is intended for the end-user (e.g., displaying stats on a screen or inventory UI).
- **Triggered By:** `print(obj)`, `str(obj)`, or formatting with f-strings `f"{obj}"`.

**Input:**
```python
class FreeFireWeapon:
    def __init__(self, name, ammo, damage=150):
        self.name = name
        self.ammo = ammo
        self.damage = damage

    # Informal string representation for the user/player
    def __str__(self):
        return f"🔫 Weapon: {self.name} | Ammo: {self.ammo} | Base Damage: {self.damage}"

weapon = FreeFireWeapon("AWM", 5)

# This now calls __str__ automatically!
print(weapon)
print(str(weapon))
```
**Output:**
```text
🔫 Weapon: AWM | Ammo: 5 | Base Damage: 150
🔫 Weapon: AWM | Ammo: 5 | Base Damage: 150
```

---

## 3. The `__repr__` Method
The `__repr__` method returns an unambiguous, formal string representation of the object. It is intended for developers, debugging, and logging. 
- **Goal:** Ideally, the string returned by `__repr__` should look like a valid Python expression that can be copied and pasted to recreate the exact same object.
- **Triggered By:** `repr(obj)`, interactive REPL inspection, and when objects are nested inside collections (like lists).

**Input:**
```python
class FreeFireWeapon:
    def __init__(self, name, ammo, damage=150):
        self.name = name
        self.ammo = ammo
        self.damage = damage

    # Formal string representation for developers
    def __repr__(self):
        return f"FreeFireWeapon(name='{self.name}', ammo={self.ammo}, damage={self.damage})"

weapon = FreeFireWeapon("AWM", 5)

# Get formal representation
print("Developer Repr:", repr(weapon))
```
**Output:**
```text
Developer Repr: FreeFireWeapon(name='AWM', ammo=5, damage=150)
```

---

## 4. ⚠️ CRITICAL GOTCHA: Objects Inside Collections
A very common Python gotcha occurs when you put objects inside a list, tuple, or dictionary. 

Even if you have defined a beautiful `__str__` method, Python will **default to calling `__repr__`** when printing the collection! If you don't define `__repr__`, you will get a list of unreadable memory addresses.

Let's see this in action:

**Input:**
```python
class WeaponWithoutRepr:
    def __init__(self, name):
        self.name = name
    def __str__(self):
        return self.name

class WeaponWithBoth:
    def __init__(self, name):
        self.name = name
    def __str__(self):
        return self.name
    def __repr__(self):
        return f"Weapon({self.name!r})"  # !r formats it with quotes (representation)

# 1. Printing an inventory list of weapons WITHOUT __repr__
inventory_a = [WeaponWithoutRepr("AWM"), WeaponWithoutRepr("M1887")]
print("Inventory A (No Repr):", inventory_a)

# 2. Printing an inventory list of weapons WITH __repr__
inventory_b = [WeaponWithBoth("AWM"), WeaponWithBoth("M1887")]
print("Inventory B (With Repr):", inventory_b)
```
**Output:**
```text
Inventory A (No Repr): [<__main__.WeaponWithoutRepr object at 0x...>, <__main__.WeaponWithoutRepr object at 0x...>]
Inventory B (With Repr): [Weapon('AWM'), Weapon('M1887')]
```

> [!TIP]
> **Best Practice:** Always define at least `__repr__` for your classes. If a class has a `__repr__` but no `__str__`, Python will fall back to using `__repr__` when you print the object or call `str()`.

---

## 5. Summary: `__str__` vs. `__repr__`

| Feature | `__str__` | `__repr__` |
| :--- | :--- | :--- |
| **Purpose** | Readable, friendly representation for end-users. | Unambiguous representation for developers/debugging. |
| **Target Audience** | End-users / CLI UI / Players. | Developers / Logs / Debuggers. |
| **Fallback** | If missing, Python calls `__repr__`. | If missing, Python uses default memory reference. |
| **Collections behavior** | Not used when printing lists/dicts. | Always used when printing lists/dicts of objects. |
