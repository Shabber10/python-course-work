# 1. Modules and Import System

A **module** is simply a file containing Python definitions and statements. Modules allow you to break down large, complex code bases into smaller, self-contained, and reusable files.

---

## 1. Import Syntax Variations
Python provides several ways to import modules and their contents. Let's use the built-in `math` module as an example.

### A. Importing the Entire Module (`import module`)
Importing the entire module keeps its variables and functions enclosed inside the module's namespace. You must access them using the dot (`.`) syntax.

**Input:**
```python
import math

# We access items using module_name.item_name
print(math.sqrt(25))
print(math.pi)
```
**Output:**
```text
5.0
3.141592653589793
```

### B. Importing Specific Names (`from module import name`)
This imports specific variables or functions directly into your current namespace. You do not need to use the module name prefix.

**Input:**
```python
from math import sqrt, pi

# Access directly
print(sqrt(25))
print(pi)
```
**Output:**
```text
5.0
3.141592653589793
```

### C. Importing with an Alias (`import module as alias`)
You can rename a module during import to shorten its name or prevent naming conflicts.

**Input:**
```python
import math as m

print(m.sqrt(16))
```
**Output:**
```text
4.0
```

### D. Importing Specific Names with an Alias
**Input:**
```python
from math import sqrt as square_root

print(square_root(9))
```
**Output:**
```text
3.0
```

### E. Importing Everything (`from module import *`)
This imports all functions, variables, and classes from the target module directly into your current namespace.

> [!CAUTION]
> Using `import *` is considered **bad practice** in professional Python development. It pollutes your local namespace, making it hard to trace where a function came from, and can accidentally overwrite local variables with the same name.

---

## 2. Finding Available Names: `dir()`
The built-in `dir()` function returns a sorted list of strings containing the names defined by a module.

**Input:**
```python
import math

# Print first 5 names defined in math module
print(dir(math)[:5])
```
**Output:**
```text
['__doc__', '__loader__', '__name__', '__package__', '__spec__']
```

---

## 3. Module Search Path: `sys.path`
When you import a module (e.g. `import my_module`), the Python interpreter searches for a file named `my_module.py` in the following locations in order:
1. The **current directory** from which the script is run.
2. The list of directories defined in the **`PYTHONPATH`** environment variable (if set).
3. The installation-dependent **standard library** directories (built-in modules).

You can inspect the exact search directories of your active environment by printing `sys.path`:

**Input:**
```python
import sys

# print the first path (usually current working directory)
print("First search directory:", sys.path[0])
```
**Output:**
```text
First search directory: c:\Users\SHABBER HUSSAIN\Desktop\python-course-work
```
