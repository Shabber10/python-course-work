# 1. Modules and Import System

A **module** is simply a file containing Python definitions and statements. Modules allow you to break down large, complex code bases into smaller, self-contained, and reusable files.

---

## 1. Import Syntax Variations
Python provides several ways to import modules and their contents. Let's use the built-in `math` module as an example.

### A. Importing the Entire Module (`import module`)
Importing the entire module keeps its variables and functions enclosed inside the module's namespace. You must access them using the dot (`.`) syntax.

**Input:**
```python
import math

# We access items using module_name.item_name
print(math.sqrt(25))
print(math.pi)
```
**Output:**
```text
5.0
3.141592653589793
```

### B. Importing Specific Names (`from module import name`)
This imports specific variables or functions directly into your current namespace. You do not need to use the module name prefix.

**Input:**
```python
from math import sqrt, pi

# Access directly
print(sqrt(25))
print(pi)
```
**Output:**
```text
5.0
3.141592653589793
```

### C. Importing with an Alias (`import module as alias`)
You can rename a module during import to shorten its name or prevent naming conflicts.

**Input:**
```python
import math as m

print(m.sqrt(16))
```
**Output:**
```text
4.0
```

### D. Importing Specific Names with an Alias
**Input:**
```python
from math import sqrt as square_root

print(square_root(9))
```
**Output:**
```text
3.0
```

### E. Importing Everything (`from module import *`)
This imports all functions, variables, and classes from the target module directly into your current namespace.

> [!CAUTION]
> Using `import *` is considered **bad practice** in professional Python development. It pollutes your local namespace, making it hard to trace where a function came from, and can accidentally overwrite local variables with the same name.

---

## 2. Module Caching and Reloading
When you import a module for the first time, Python runs all the code in that module and caches the compiled module object inside the **`sys.modules`** dictionary. 

If you call `import my_module` a second time in the same script, Python will **not** execute the module code again. Instead, it reads the module object directly from the cache.

### The Reload Gotcha
If your module's code changes dynamically (e.g. during an interactive Jupyter session or live configuration edits) and you want to reload the module without restarting Python, a standard `import` will do nothing because of the cache. 

You must use the **`importlib.reload()`** function to force a reload.

**Input:**
```python
import importlib
import my_helper  # First import runs module code

# ... imagine my_helper.py file gets updated on disk ...

# To force Python to reload the updated code:
importlib.reload(my_helper)
```

## 3. Finding Available Names: `dir()`
The built-in `dir()` function returns a sorted list of strings containing the names defined by a module.

**Input:**
```python
import math

# Print first 5 names defined in math module
print(dir(math)[:5])
```
**Output:**
```text
['__doc__', '__loader__', '__name__', '__package__', '__spec__']
```

## 4. The `sys` and `platform` Modules

Python provides built-in modules to interact directly with the Python interpreter runtime and the underlying operating system.

### A. The `sys` Module
The `sys` module provides variables and functions that interact closely with the Python interpreter.

- **`sys.version`**: A string containing the version number of the active Python interpreter.
- **`sys.argv`**: A list of command-line arguments passed to a Python script. Index `0` is the script name itself.
- **`sys.path`**: A list of directory paths where Python searches for modules during imports.
- **`sys.exit()`**: Exits the Python interpreter immediately, stopping all execution.

**Input:**
```python
import sys

# Get Python Version
print("Python Version:", sys.version)

# Print search paths
print("First search directory:", sys.path[0])

# Print arguments list (depends on how script was launched)
print("Command Line Arguments:", sys.argv)

print("Before Exit")
sys.exit()
print("After Exit")  # This line will NEVER run!
```
**Output:**
```text
Python Version: 3.10.5 (tags/v3.10.5:f377153, Jun  6 2022, 16:14:13)
First search directory: c:\Users\SHABBER HUSSAIN\Desktop\python-course-work
Command Line Arguments: ['my_script.py']
Before Exit
```

### B. The `platform` Module
The `platform` module allows you to retrieve information about the operating system, hardware, and processor details on which Python is running.

- **`platform.system()`**: Returns the OS name (e.g., `'Windows'`, `'Linux'`, or `'Darwin'` for macOS).
- **`platform.release()`**: Returns the OS release version (e.g., `'10'` or `'22.04'`).
- **`platform.processor()`**: Returns the CPU/hardware processor type.

**Input:**
```python
import platform

print("OS System:", platform.system())
print("OS Release:", platform.release())
print("Processor:", platform.processor())
```
**Output (System Dependent):**
```text
OS System: Windows
OS Release: 10
Processor: Intel64 Family 6 Model 158 Stepping 10, GenuineIntel
```
