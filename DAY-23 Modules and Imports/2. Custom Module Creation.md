# 2. Custom Module Creation

Creating your own module is as simple as saving Python code in a file with a `.py` extension. You can then import that file in any other Python script located in the same directory.

---

## 1. Writing a Custom Module
Let us create a file named `my_helper.py` and write some functions and a variable:

### `my_helper.py`
```python
# Variables in a module
course_name = "Python Masterclass"

# Functions in a module
def square(x):
    return x * x

def greet(name):
    return f"Hello, {name}!"
```

---

## 2. Importing a Custom Module
Let's import our helper module inside another script (e.g. `main.py` located in the same folder).

**Input:**
```python
# Import the custom module
import my_helper

# Using variables and functions from the custom module
print("Course:", my_helper.course_name)
print("Square of 5:", my_helper.square(5))
print("Greeting:", my_helper.greet("Alice"))
```
**Output:**
```text
Course: Python Masterclass
Square of 5: 25
Greeting: Hello, Alice!
```

---

## 3. Namespace Pollution and Collisions
When you import a custom module, Python keeps all its items nested inside the module's namespace (accessible via `my_helper.item`). This is called **Namespace Isolation**.

If you import functions directly using `from my_helper import *` or `from my_helper import square`, you bring those names into your local scope. If you already have a local function named `square`, a **Namespace Collision** occurs: the last defined or imported name will overwrite (shadow) the earlier one!

### Example of a Collision:
```python
def square(x):
    return "This is a custom local shape square function!"

from my_helper import square  # Overwrites our local 'square' function!

print(square(5))  # Outputs 25, instead of printing our local text!
```

> [!TIP]
> To prevent namespace pollution, prefer `import my_helper` (keeps namespaces separate) or use aliases `from my_helper import square as helper_square`.

## 4. The `if __name__ == "__main__":` Block
When a Python file is executed, Python sets a special built-in variable named `__name__` before running the code.

- If the file is being **run directly** (e.g., `python my_helper.py`), `__name__` is set to `"__main__"`.
- If the file is being **imported** as a module in another script (e.g., `import my_helper`), `__name__` is set to the name of the file (in this case, `"my_helper"`).

We use `if __name__ == "__main__":` to write code (like tests or demonstration calls) that should **only run when executed directly**, and should be ignored when imported.

### Example:
Let's modify `my_helper.py`:

```python
course_name = "Python Masterclass"

def square(x):
    return x * x

# This block only runs when you execute 'python my_helper.py'
if __name__ == "__main__":
    print("--- Running tests for my_helper ---")
    assert square(4) == 16, "Test Failed!"
    print("Test passed! square(4) is indeed 16.")
```

#### If we run `my_helper.py` directly:
`python my_helper.py`
**Output:**
```text
--- Running tests for my_helper ---
Test passed! square(4) is indeed 16.
```

#### If we import `my_helper` in `main.py`:
`import my_helper`
**Output:**
```text
(Nothing is printed, because __name__ is "my_helper", so the test block is skipped!)
```

## 5. Practical Application: Building an ATM System
Let's apply custom module concepts to build a modular ATM terminal program. We split the code into two files:
1. `atm.py`: Manages the account balance state and transaction logic.
2. `main.py`: Handles the user interface terminal loop and imports `atm`.

### File 1: `atm.py`
```python
balance = 10000  # Global variable holding account state

def check_balance():
    print("Available Balance:", balance)

def deposit(amount):
    global balance
    balance += amount
    print("Amount Deposited Successfully")
    print("Current Balance:", balance)

def withdraw(amount):
    global balance
    if amount <= balance:
        balance -= amount
        print("Please collect your cash.")
        print("Current Balance:", balance)
    else:
        print("Insufficient Balance")
```

### File 2: `main.py`
```python
import atm

while True:
    print("\n===== ATM MENU =====")
    print("1. Check Balance")
    print("2. Deposit")
    print("3. Withdraw")
    print("4. Exit")

    choice = int(input("Enter your choice: "))

    if choice == 1:
        atm.check_balance()
    elif choice == 2:
        amount = int(input("Enter amount to deposit: "))
        atm.deposit(amount)
    elif choice == 3:
        amount = int(input("Enter amount to withdraw: "))
        atm.withdraw(amount)
    elif choice == 4:
        print("Thank you for using our ATM!")
        break
    else:
        print("Invalid Choice. Please try again.")
```
