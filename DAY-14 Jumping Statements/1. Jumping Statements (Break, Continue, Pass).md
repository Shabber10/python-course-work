# 1. Jumping Statements (Break, Continue, Pass)

Jumping statements allow you to alter the normal execution flow of loops (both `while` loops and `for` loops). They give you the power to terminate a loop entirely, or to skip specific iterations.

---

## 1. The `break` Statement
With the `break` statement we can absolutely stop the loop, even if the `while` condition is true, or even if the `for` loop has not finished looping through the sequence.

### A. Break in a `while` loop
**Input:**
```python
i = 1
while i < 6:
    print(i)
    if i == 3:
        print("Breaking the loop at 3!")
        break
    i += 1
```
**Output:**
```text
1
2
3
Breaking the loop at 3!
```

### B. Break in a `for` loop
**Input:**
```python
fruits = ["apple", "banana", "cherry"]
for x in fruits:
    print(x)
    if x == "banana":
        print("Found the banana! Stopping search.")
        break
```
**Output:**
```text
apple
banana
Found the banana! Stopping search.
```

---

## 2. The `continue` Statement
With the `continue` statement we can **skip the current iteration** of the loop, and immediately jump back to the top to start the next iteration. It does NOT destroy the loop like `break` does!

### A. Continue in a `while` loop
**Input:**
```python
i = 0
while i < 6:
    i += 1
    if i == 3:
        # Skips printing the number 3
        continue
    print(i)
```
**Output:**
```text
1
2
4
5
6
```

### B. Continue in a `for` loop
**Input:**
```python
fruits = ["apple", "banana", "cherry", "kiwi"]

for x in fruits:
    if x == "banana" or x == "cherry":
        continue # Skip banana and cherry!
    print("Fruit:", x)
```
**Output:**
```text
Fruit: apple
Fruit: kiwi
```

---

## 3. The `pass` Statement
`for` loops cannot be empty. If you have a `for` loop with no content (maybe you are waiting for another programmer to finish that part), Python will throw an error. 

Just like with `if` statements, you can put in the `pass` statement as a temporary placeholder to avoid getting an error.

**Input:**
```python
# The pass statement does literally nothing, it just acts as a placeholder
for x in [0, 1, 2]:
    pass

print("Program ran successfully!")
```
**Output:**
```text
Program ran successfully!
```
