# 2. Python Syntax and Strict Indentation

Unlike many other popular programming languages (like C++, Java, or JavaScript) that use curly brackets `{}` or specific keywords to define blocks of code, Python relies entirely on **Indentation**. 

Indentation refers to the spaces at the beginning of a code line. Where in other languages the indentation is just for readability, in Python, **the indentation is a strict requirement for the code to run.**

---

## 1. How Indentation Works

A block of code (such as the code inside an `if` statement or a `for` loop) is denoted by line indentation. All lines of code within the same block must have the exact same number of spaces.

**Input:**
```python
if 5 > 2:
    print("Five is greater than two!") # Indented correctly
    print("This line belongs to the 'if' block.")
```
**Output:**
```text
Five is greater than two!
This line belongs to the 'if' block.
```

---

## 2. Common Errors (IndentationError)

If you forget to indent, Python will throw an `IndentationError` and your program will crash. This forces programmers to write clean, easily readable code.

**Input:**
```python
if 5 > 2:
print("Five is greater than two!") # ERROR: Missing indentation
```
**Output:**
```text
  File "main.py", line 2
    print("Five is greater than two!")
    ^
IndentationError: expected an indented block
```

### Inconsistent Indentation
You have to use the exact same number of spaces in the same block of code, otherwise Python will give you an error. The standard convention in Python is to use **4 spaces** for each indentation level, but you can use 2, 3, or more, as long as you are consistent.

**Input:**
```python
if 5 > 2:
    print("Five is greater than two!")
      print("This has too many spaces!") # ERROR: Inconsistent
```
**Output:**
```text
  File "main.py", line 3
    print("This has too many spaces!")
    ^
IndentationError: unexpected indent
```

---

## 3. Advanced Example: Nested Indentation

When you have blocks of code inside of other blocks of code (called "nesting"), you must indent further to the right. Each level of nesting adds another 4 spaces.

**Input:**
```python
age = 20
has_ticket = True

if age >= 18:                      # Level 0 (No spaces)
    print("You are an adult.")     # Level 1 (4 spaces)
    if has_ticket:                 # Level 1 (4 spaces)
        print("You can enter!")    # Level 2 (8 spaces)
    else:                          # Level 1 (4 spaces)
        print("Buy a ticket.")     # Level 2 (8 spaces)
```
**Output:**
```text
You are an adult.
You can enter!
```

---

## 4. Best Practices (PEP 8)

PEP 8 is the official style guide for Python code. Regarding syntax and indentation, PEP 8 states:
1. **Use 4 spaces per indentation level.**
2. **Never mix tabs and spaces.** Even if they look identical visually in your code editor, Python treats a Tab character differently than 4 Space characters, which will cause confusing errors. Modern code editors automatically convert your Tab key presses into 4 Spaces.
