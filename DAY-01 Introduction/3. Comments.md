# 3. Comments in Python

Comments are lines in your code that the Python interpreter completely ignores. They are used exclusively for human programmers to read. 

Using comments effectively is one of the most important skills for a programmer, especially when working on teams or returning to your own code months later!

---

## 1. Why Use Comments?
1. **Explain Python Code:** To explain what a complex block of code is doing in plain English.
2. **Make the Code More Readable:** To separate sections of a massive file.
3. **Prevent Execution:** To disable certain lines of code temporarily while testing or debugging.

---

## 2. Single-Line Comments (`#`)

Python uses the hash symbol `#` to denote a single-line comment. Everything on the line after the `#` will be ignored by Python.

**Input:**
```python
# This entire line is a comment and will be ignored
print("Hello, World!") # This is an inline comment explaining the code
```
**Output:**
```text
Hello, World!
```

---

## 3. Multi-Line Comments

Python does not actually have a dedicated syntax for multi-line comments like other languages do (e.g., `/* comment */` in C++). However, there are two common ways to achieve this.

### A. Multiple Hash Symbols
The most common way is to simply insert a `#` for each line.

**Input:**
```python
# This is a comment
# written in
# more than just one line
print("Executing code!")
```
**Output:**
```text
Executing code!
```

### B. Unassigned String Literals (Docstrings)
Because Python ignores string literals that are not assigned to a variable, you can use triple quotes `"""` to write a multi-line string. If it is not assigned to anything, Python will read the code, but then immediately ignore it. This is widely used for multi-line comments.

**Input:**
```python
"""
This is a multi-line string.
It serves as a multi-line comment in Python!
You can write as many lines as you want here.
"""
print("Docstrings are cool!")
```
**Output:**
```text
Docstrings are cool!
```

---

## 4. Disabling Code (Commenting Out)

If your code is broken and you want to test exactly which line is causing the issue, you don't need to delete the code! You can simply "comment it out" so Python skips over it.

**Input:**
```python
print("Test 1 passed")
# print("Test 2 might fail, so I commented it out")
print("Test 3 passed")
```
**Output:**
```text
Test 1 passed
Test 3 passed
```

---

## 5. Best Practices
1. **Don't State the Obvious:** Avoid writing comments that simply repeat what the code obviously does.
   * *Bad:* `# Adds 1 to x` (for the code `x += 1`)
   * *Good:* `# Increment score for passing the checkpoint`
2. **Keep Comments Updated:** If you change the code, ALWAYS remember to update the comment! A wrong comment is worse than no comment.
3. **Use Docstrings for Functions:** Always use the `"""` syntax to describe what a function does at the very top of the function definition.
