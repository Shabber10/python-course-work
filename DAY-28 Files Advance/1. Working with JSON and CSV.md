# 1. Working with JSON and CSV

Plain text files are excellent for simple logs, but they are difficult to query or parse when storing complex records. For structured information, we use industry-standard data formats. The two most common in Python development are **JSON** (JavaScript Object Notation) and **CSV** (Comma-Separated Values).

---

## 1. Working with JSON

JSON is the standard data format for web APIs, configuration files, and database exports. It maps directly to Python's built-in dictionaries and lists.

Python provides the built-in `json` module to handle these conversions:
- **Serialization (Encoding):** Converting a Python dictionary/list to a JSON string or file.
- **Deserialization (Decoding):** Parsing a JSON string or file back into a Python dictionary/list.

```text
               JSON CONVERSION MAP:
  
      Python Dictionary                JSON String / File
     ┌─────────────────┐  json.dump()  ┌──────────────────┐
     │ {'name': 'Sam'} │ ────────────> │ {"name": "Sam"}  │
     │                 │ <──────────── │                  │
     └─────────────────┘  json.load()  └──────────────────┘
```

### A. Deserialization (Reading JSON)
- `json.loads(string)`: Parses a JSON-formatted **string** into a Python dictionary.
- `json.load(file)`: Reads a JSON-formatted **file** and parses it into a Python dictionary.

**Input:**
```python
import json

# 1. Parsing from a string (loads = "load string")
json_payload = '{"username": "johndoe", "active": true, "roles": ["Admin", "User"]}'
user_dict = json.loads(json_payload)

print("Type of user_dict:", type(user_dict))
print("Roles list      :", user_dict["roles"])

# 2. Parsing from a file (load)
# Let's create a temporary json file first
with open("user.json", "w") as f:
    f.write('{"name": "Emma", "status": "Active"}')

with open("user.json", "r") as file:
    data = json.load(file)
    print("Read from file  :", data["name"], "is", data["status"])
```
**Output:**
```text
Type of user_dict: <class 'dict'>
Roles list      : ['Admin', 'User']
Read from file  : Emma is Active
```

### B. Serialization (Writing JSON)
- `json.dumps(dict)`: Converts a Python dictionary into a JSON-formatted **string**.
- `json.dump(dict, file)`: Converts and writes a Python dictionary to a **file**.
- **Pretty Printing:** Pass `indent=4` to format the JSON with clean indentation. Pass `sort_keys=True` to sort the keys alphabetically.

**Input:**
```python
import json

config_data = {
    "resolution": "1920x1080",
    "volume": 80,
    "fullscreen": False
}

# Serialize to String with formatting parameters
json_string = json.dumps(config_data, indent=4, sort_keys=True)
print("--- Formatted JSON String ---")
print(json_string)

# Serialize directly to a file
with open("config.json", "w") as file:
    json.dump(config_data, file, indent=4)
    print("📝 Configuration successfully saved to config.json.")
```
**Output:**
```text
--- Formatted JSON String ---
{
    "fullscreen": false,
    "resolution": "1920x1080",
    "volume": 80
}
📝 Configuration successfully saved to config.json.
```

## 2. Working with CSV

CSV files store tabular data (spreadsheets) as plaintext lines, where columns are separated by commas (or other delimiters like semicolons).

Python's built-in `csv` module provides tools to read and write CSV files.

### A. Reading CSV files
You can read CSV files as **lists** of strings, or as **dictionaries** where column headers automatically become dictionary keys.

#### 1. Using `csv.reader` (Returns Lists)
Each row is read as a list of string values.

**Input:**
```python
import csv

# Let's create a temporary CSV file
with open("customers.csv", "w", newline="") as f:
    f.write("Name,City,Age\nAlice,Seattle,28\nBob,Boston,32\n")

with open("customers.csv", "r") as file:
    reader = csv.reader(file)
    header = next(reader)  # Read the first header row
    print("CSV Header:", header)
    
    for row in reader:
        print(f"Customer Name: {row[0]} | City: {row[1]} | Age: {row[2]}")
```
**Output:**
```text
CSV Header: ['Name', 'City', 'Age']
Customer Name: Alice | City: Seattle | Age: 28
Customer Name: Bob | City: Boston | Age: 32
```

#### 2. Using `csv.DictReader` (Returns Dictionaries - Recommended!)
This maps each row directly to a dictionary, using the first row values as the keys. This is much cleaner and more readable, as you do not need to remember column indices (`row[0]`, `row[1]`).

**Input:**
```python
import csv

with open("customers.csv", "r") as file:
    # DictReader automatically parses headers
    reader = csv.DictReader(file)
    for row in reader:
        print(f"Customer: {row['Name']} lives in {row['City']} (Age: {row['Age']})")
```
**Output:**
```text
Customer: Alice lives in Seattle (Age: 28)
Customer: Bob lives in Boston (Age: 32)
```

### B. Writing CSV files
Similar to reading, you can write CSVs from lists of rows or from dictionaries.

#### 1. Using `csv.writer` (Writes Lists)
**Input:**
```python
import csv

data_to_write = [
    ["ID", "Product", "Price"],
    [101, "Keyboard", 79.99],
    [102, "Mouse", 39.99]
]

# Note: Always pass newline="" to open() when writing CSVs to prevent blank lines on Windows
with open("products.csv", "w", newline="") as file:
    writer = csv.writer(file)
    writer.writerows(data_to_write)  # Writes all rows at once
```

#### 2. Using `csv.DictWriter` (Writes Dictionaries - Recommended!)
Requires you to specify the column headers (`fieldnames`) explicitly.

**Input:**
```python
import csv

headers = ["Name", "Score", "Grade"]
students = [
    {"Name": "Sam", "Score": 95, "Grade": "A"},
    {"Name": "Lisa", "Score": 88, "Grade": "B"}
]

with open("scores.csv", "w", newline="") as file:
    # Set up DictWriter
    writer = csv.DictWriter(file, fieldnames=headers)
    
    # 1. Write the header row
    writer.writeheader()
    
    # 2. Write rows
    writer.writerows(students)

# Verify scores.csv content
with open("scores.csv", "r") as file:
    print("--- Generated CSV File ---")
    print(file.read())
```
**Output:**
```text
--- Generated CSV File ---
Name,Score,Grade
Sam,95,A
Lisa,88,B
```
