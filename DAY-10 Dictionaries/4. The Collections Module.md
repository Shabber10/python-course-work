# 4. The Collections Module

Python's built-in container types—**Lists**, **Tuples**, **Sets**, and **Dictionaries**—are powerful and cover 90% of your programming needs. However, there are specialized situations where you need custom behavior:
- Automatically counting occurrences of elements.
- Initializing missing dictionary keys without crashing.
- Appending or popping elements from the start of a list in $O(1)$ time.
- Referencing tuple elements by name instead of indices.

To handle these scenarios elegantly and efficiently, Python provides the **`collections`** module. It contains specialized container datatypes that act as high-performance alternatives to general-purpose built-ins.

---

## 1. `Counter` (The Frequency Counter)
Writing a manual loop to count the frequency of elements in a list or string is tedious and requires several lines of logic:

```python
# The manual way
counts = {}
for word in words:
    if word not in counts:
        counts[word] = 1
    else:
        counts[word] += 1
```

`Counter` simplifies this entire operation into a **single line of code**. It is a subclass of the standard dictionary designed specifically for counting hashable objects.

### A. Basic Counting and Accessing
When you pass an iterable (like a list or string) into `Counter`, it returns a dictionary-like object mapping elements to their frequency counts.

**Input:**
```python
from collections import Counter

# Let's count occurrences of items in a list
fruits = ["apple", "banana", "apple", "cherry", "banana", "apple"]
fruit_counter = Counter(fruits)

print("Counter Object:", fruit_counter)

# Accessing counts
print("Count of 'apple':", fruit_counter["apple"])
print("Count of 'cherry':", fruit_counter["cherry"])

# Accessing a missing element does NOT raise a KeyError! It returns 0.
print("Count of 'mango':", fruit_counter["mango"])
```
**Output:**
```text
Counter Object: Counter({'apple': 3, 'banana': 2, 'cherry': 1})
Count of 'apple': 3
Count of 'cherry': 1
Count of 'mango': 0
```

### B. Finding Common Elements: `.most_common(n)`
A common problem in data analysis is finding the "top $n$" items. `Counter` handles this with the `.most_common(n)` method, which returns a list of tuples containing the highest frequency items.

**Input:**
```python
# Get the top 2 most frequent fruits
top_two = fruit_counter.most_common(2)
print("Top 2 Fruits:", top_two)

# If n is omitted, it returns all elements sorted by frequency
print("All sorted:", fruit_counter.most_common())
```
**Output:**
```text
Top 2 Fruits: [('apple', 3), ('banana', 2)]
All sorted: [('apple', 3), ('banana', 2), ('cherry', 1)]
```

### C. Arithmetic Operations with `Counter`
You can combine multiple `Counter` objects using mathematical operators like addition (`+`), subtraction (`-`), intersection (`&`), and union (`|`).

> [!NOTE]
> Subtracting counters will only keep positive result values. Any values that become `0` or negative are omitted from the result.

**Input:**
```python
counter_a = Counter(x=3, y=1)
counter_b = Counter(x=1, y=2)

print("Addition (+):", counter_a + counter_b)        # Adds frequencies together
print("Subtraction (-):", counter_a - counter_b)     # Subtracts frequencies
print("Intersection (&):", counter_a & counter_b)    # Minimum of common counts
print("Union (|):", counter_a | counter_b)           # Maximum of common counts
```
**Output:**
```text
Addition (+): Counter({'x': 4, 'y': 3})
Subtraction (-): Counter({'x': 2})
Intersection (&): Counter({'x': 1, 'y': 1})
Union (|): Counter({'x': 3, 'y': 2})
```

## 2. `defaultdict` (The KeyError Preventer)
When working with standard dictionaries, attempting to access or modify a key that doesn't exist yet will raise a `KeyError`. You are forced to write checks:

```python
# The manual way to group items
grouped = {}
for student, grade in grades:
    if student not in grouped:
        grouped[student] = []
    grouped[student].append(grade)
```

`defaultdict` solves this by allowing you to specify a **default factory function** (such as `list`, `int`, `set`, or `dict`) when creating the dictionary. When a missing key is accessed, `defaultdict` automatically calls the factory function to initialize the key with a default value.

### A. Grouping Elements using `defaultdict(list)`
Passing `list` as the factory function initializes missing keys to an empty list `[]`.

**Input:**
```python
from collections import defaultdict

# We want to group cities by country
countries_data = [("USA", "New York"), ("Canada", "Toronto"), ("USA", "Chicago"), ("Canada", "Vancouver")]

# Non-existent keys automatically initialize to list() -> []
country_map = defaultdict(list)

for country, city in countries_data:
    # No need to check "if country not in country_map"!
    country_map[country].append(city)

# Convert to standard dict for clean printing
print("Grouped Dictionary:", dict(country_map))
```
**Output:**
```text
Grouped Dictionary: {'USA': ['New York', 'Chicago'], 'Canada': ['Toronto', 'Vancouver']}
```

### B. Counting Items using `defaultdict(int)`
Passing `int` as the factory function initializes missing keys to `0` (the default value of an integer).

**Input:**
```python
text = "hello"
char_counts = defaultdict(int)

for char in text:
    # Accessing a new character initializes it to 0, then we add 1
    char_counts[char] += 1

print("Character counts:", dict(char_counts))
```
**Output:**
```text
Character counts: {'h': 1, 'e': 1, 'l': 2, 'o': 1}
```

## 3. `deque` (Double-Ended Queue)
Regular Python lists are highly optimized for fast $O(1)$ operations at the **end** (like `append()` and `pop()`). However, inserting or deleting an item at the **beginning** (like `insert(0, value)` or `pop(0)`) takes linear time $O(n)$ because Python has to shift all subsequent elements in memory by one position.

A `deque` (Double-Ended Queue, pronounced "deck") is implemented as a **doubly linked list**. It provides $O(1)$ constant-time insertion and deletion from **both** the left and right ends.

| Operation | Regular List | `deque` |
| :--- | :---: | :---: |
| Append to Right | $O(1)$ | $O(1)$ |
| Pop from Right | $O(1)$ | $O(1)$ |
| Insert at Left (index 0) | $O(n)$ (Slow) | $O(1)$ (Fast!) |
| Pop from Left (index 0) | $O(n)$ (Slow) | $O(1)$ (Fast!) |

### A. Deque Appends and Pops
We use `appendleft()` and `popleft()` to perform fast operations on the left side of the queue.

**Input:**
```python
from collections import deque

d = deque(["center"])

# Adding elements
d.append("right")       # Adds to right end
d.appendleft("left")    # Adds to left end
print("After appends:", d)

# Removing elements
right_item = d.pop()      # Removes from right end
left_item = d.popleft()   # Removes from left end
print(f"Popped items: Left={left_item}, Right={right_item}")
print("Remaining deque:", d)
```
**Output:**
```text
After appends: deque(['left', 'center', 'right'])
Popped items: Left=left, Right=right
Remaining deque: deque(['center'])
```

### B. Limiting Size: `maxlen`
You can create a `deque` with a fixed maximum size using the `maxlen` parameter. When the deque becomes full and new items are added, elements from the opposite end are automatically discarded. This is perfect for maintaining a **moving window** of recent data.

**Input:**
```python
# Create a deque that holds at most 3 items
history = deque(maxlen=3)

history.append("page1")
history.append("page2")
history.append("page3")
print("Full history:", history)

# Adding a 4th page pushes "page1" out from the left!
history.append("page4")
print("Updated history:", history)
```
**Output:**
```text
Full history: deque(['page1', 'page2', 'page3'], maxlen=3)
Updated history: deque(['page2', 'page3', 'page4'], maxlen=3)
```

## 4. `namedtuple` (Self-Documenting Tuples)
Standard tuples are lightweight and immutable, which makes them great for grouping data. However, referencing fields by numeric index makes code hard to read and write:

```python
# Regular tuple - what do index 0 and 1 represent?
point = (10, 20)
print(point[0], point[1])
```

`namedtuple` is a factory function that generates a subclass of a tuple where each index has a name. You can access values using both **index position** and **descriptive field names** (dot notation).

### A. Creating and Accessing Named Tuples
To create a named tuple, define the name of the tuple subclass and a list of field names.

**Input:**
```python
from collections import namedtuple

# Define a named tuple subclass 'Point' with fields 'x' and 'y'
Point = namedtuple("Point", ["x", "y"])

# Instantiating the named tuple
p = Point(x=10, y=20)

# Accessing fields
print("Access by name (p.x):", p.x)
print("Access by name (p.y):", p.y)

# Accessing by index still works!
print("Access by index (p[0]):", p[0])

# Unpacking still works!
x_coord, y_coord = p
print(f"Unpacked Coordinates: {x_coord}, {y_coord}")
```
**Output:**
```text
Access by name (p.x): 10
Access by name (p.y): 20
Access by index (p[0]): 10
Unpacked Coordinates: 10, 20
```

### B. Conversion methods
Named tuples provide helper methods starting with an underscore `_` to prevent namespace collisions with field names.

**Input:**
```python
# Convert namedtuple to an OrderedDict
print("As Dictionary:", p._asdict())

# Create a new instance replacing specific values
new_point = p._replace(x=100)
print("Replaced Point:", new_point)
```
**Output:**
```text
As Dictionary: {'x': 10, 'y': 20}
Replaced Point: Point(x=100, y=20)
```

> [!WARNING]
> Since `namedtuple` inherits from tuple, it is **immutable**. You cannot modify its attributes directly (e.g. typing `p.x = 5` will raise an `AttributeError`). Use the `_replace()` method to create a new instance with the updated value instead.
