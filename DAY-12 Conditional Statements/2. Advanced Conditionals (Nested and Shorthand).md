# 2. Advanced Conditionals (Nested and Shorthand)

Once you understand basic `if` statements, you can use advanced techniques like nesting them inside of each other, or condensing them into a single line of code.

---

## 1. Nested `if` Statements
You can have `if` statements inside `if` statements. This is called **nesting**. 

When you nest `if` statements, you MUST indent the inner statements by another 4 spaces so Python knows they belong to the inner block!

**Input:**
```python
x = 41

if x > 10:
    print("x is above ten,")
    
    # This if statement is nested INSIDE the first one!
    if x > 20:
        print("and also above 20!")
    else:
        print("but not above 20.")
```
**Output:**
```text
x is above ten,
and also above 20!
```

---

## 2. Short Hand `if` (One Line `if`)
If you have only one statement to execute, you can put it on the same line as the `if` statement. This saves vertical space in your code!

**Input:**
```python
a = 200
b = 33

if a > b: print("a is greater than b")
```
**Output:**
```text
a is greater than b
```

---

## 3. Short Hand `if...else` (Ternary Operator)
If you have exactly one statement to execute for `if`, and one for `else`, you can put it all on the same line. 

This technique is formally known as a **Ternary Operator**, or Conditional Expression.

**Syntax:** `action_if_true if condition else action_if_false`

**Input:**
```python
a = 2
b = 330

# The print happens BEFORE the if keyword!
print("A wins") if a > b else print("B wins")
```
**Output:**
```text
B wins
```

### Multiple `else` statements on One Line
You can actually chain multiple else statements on the same line. However, this is generally discouraged by PEP 8 because it becomes very difficult to read.

**Input:**
```python
a = 330
b = 330

print("A") if a > b else print("EQUAL") if a == b else print("B")
```
**Output:**
```text
EQUAL
```

---

## 4. The `pass` Statement
`if` statements cannot be empty. If you for some reason have an `if` statement with no content, Python will throw an error. 

To prevent this error, you can use the `pass` statement as a temporary placeholder!

**Input:**
```python
a = 33
b = 200

if b > a:
    pass # Putting this here prevents a Syntax Error while we decide what to code later!

print("Program finished successfully.")
```
**Output:**
```text
Program finished successfully.
```
