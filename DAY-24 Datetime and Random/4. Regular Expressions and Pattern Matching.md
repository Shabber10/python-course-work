# 4. Regular Expressions and Pattern Matching

In software development, we often need to search, extract, or validate specific text patterns (such as email addresses, phone numbers, or credit card configurations). Python provides the built-in **`re`** (Regular Expressions) module to handle these pattern-matching tasks.

---

## 1. What is a Regular Expression (Regex)?

A Regular Expression is a special sequence of characters that forms a search pattern. Instead of searching for an exact word, you search for a *type* of word (e.g., "three digits followed by a hyphen").

### Core Matching Functions:
- **`re.compile(pattern)`:** Compiles a regular expression pattern into a reusable regex object. This is highly recommended for performance when checking the same pattern multiple times.
- **`re.search(pattern, string)`:** Scans through a string, looking for the first location where the regex pattern produces a match. Returns a **Match Object** if found, or `None` if no match exists.

**Input:**
```python
import re

# 1. Compile a pattern to match one or more digits (\d+)
pattern = re.compile(r"\d+")

# 2. Search for the pattern in a string
match_result = pattern.search("Age: 25 years")

print("Match Result Object:", match_result)
if match_result:
    print("Matched Text       :", match_result.group())  # Extracted text
```
**Output:**
```text
Match Result Object: <re.Match object; span=(5, 7), match='25'>
Matched Text       : 25
```

---

## 2. Common Regex Special Characters (Symbols)

To write patterns, we use a set of special wildcards and anchors:

| Symbol | Meaning | Example |
| :--- | :--- | :--- |
| **`^`** | Indicates the **start** of a string. | `^A` matches "Apple" but not "An Apple" |
| **`$`** | Indicates the **end** of a string. | `d$` matches "red" but not "ready" |
| **`\d`** | Matches any single numerical digit (equivalent to `[0-9]`). | `\d\d` matches "45" |
| **`+`** | Matches **one or more** repetitions of the preceding character. | `\d+` matches "7" or "258" |
| **`[ ]`** | Defines a character set (matches any single character in it). | `[789]` matches 7, 8, or 9 |
| **`[^ ]`** | A caret **inside** square brackets negates the set (matches any character NOT in it). | `[^0-9]` matches any non-digit character |
| **`{n}`** | Matches exactly $n$ occurrences of the preceding pattern. | `\d{4}` matches exactly 4 digits |

## 3. Real-World Validation Programs

Below are three standard form-validation use cases featured in development testing.

### A. Mobile Number Validation
A valid mobile number must:
1. Start with the digit 7, 8, or 9.
2. Contain exactly 10 digits in total.
3. Contain no other characters or spaces.

*Pattern:* `^[789]\d{9}$`

**Input:**
```python
import re

def validate_mobile(number_str):
    # Pattern: Start with 7-9, followed by exactly 9 more digits, then end.
    pattern = r"^[789]\d{9}$"
    
    if re.search(pattern, number_str):
        print(f"📞 {number_str} is Valid")
    else:
        print(f"❌ {number_str} is Invalid")

validate_mobile("9876543210")
validate_mobile("6876543210")  # Starts with 6 (invalid)
validate_mobile("987654321")   # Only 9 digits (invalid)
```
**Output:**
```text
📞 9876543210 is Valid
❌ 6876543210 is Invalid
❌ 987654321 is Invalid
```

### B. Credit Card Number Validation
A valid bank credit card must:
1. Start with a 4, 5, or 6.
2. Contain exactly 16 digits.
3. Be written as 16 consecutive digits (`dddddddddddddddd`) OR as 4 groups of 4 digits separated by a single hyphen (`dddd-dddd-dddd-dddd`).
4. Contain no other characters (like spaces or underscores).
5. NOT contain 4 or more consecutive repeated digits (e.g. `4444` is invalid; `4-4-4-4` is also invalid because it becomes `4444` when hyphens are removed).

**Input:**
```python
import re

def validate_credit_card(card_str):
    # 1. Structure Check: starts with 4-6, either 16 digits or 4 groups of 4 separated by -
    structure_pattern = r"^[456]\d{3}-?\d{4}-?\d{4}-?\d{4}$"
    
    if not re.search(structure_pattern, card_str):
        return "Invalid"
        
    # Remove hyphens to inspect consecutive repeated digits
    clean_card = card_str.replace("-", "")
    
    # 2. Check for 4 or more consecutive repeated digits
    for i in range(len(clean_card) - 3):
        if clean_card[i] == clean_card[i+1] == clean_card[i+2] == clean_card[i+3]:
            return "Invalid"
            
    return "Valid"

print("Card 5123-4567-8912-3456:", validate_credit_card("5123-4567-8912-3456"))
print("Card 4123333678912345    :", validate_credit_card("4123333678912345"))  # 3333 is repeated (invalid)
```
**Output:**
```text
Card 5123-4567-8912-3456: Valid
Card 4123333678912345    : Invalid
```

### C. Leap Year and Calendar Date Validation (`dd/mm/yyyy`)
We want to check if a date string in `dd/mm/yyyy` format is a valid calendar date.

**Input:**
```python
import re

def is_leap_year(year):
    # Leap year condition: divisible by 4, but not by 100 unless also by 400
    return year % 4 == 0 and (year % 100 != 0 or year % 400 == 0)

def validate_date(date_str):
    # 1. Regex format check (exactly 2 digits, slash, 2 digits, slash, 4 digits)
    if not re.search(r"^\d{2}/\d{2}/\d{4}$", date_str):
        return "Invalid"
        
    day, month, year = map(int, date_str.split("/"))
    
    if month < 1 or month > 12:
        return "Invalid"
        
    # Days in each month
    days_in_month = [0, 31, 28, 31, 30, 31, 30, 31, 31, 30, 31, 30, 31]
    
    # Adjust February for leap years
    if is_leap_year(year):
        days_in_month[2] = 29
        
    if day < 1 or day > days_in_month[month]:
        return "Invalid"
        
    return "Valid"

print("Date 29/02/2024:", validate_date("29/02/2024"))  # 2024 is leap year (valid)
print("Date 29/02/2023:", validate_date("29/02/2023"))  # 2023 is not leap year (invalid)
```
**Output:**
```text
Date 29/02/2024: Valid
Date 29/02/2023: Invalid
```
