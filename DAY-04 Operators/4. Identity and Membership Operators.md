# 4. Identity and Membership Operators

Python provides two highly specific and uniquely readable sets of operators: **Identity Operators** and **Membership Operators**. These are used constantly in professional Python code.

---

## 1. Identity Operators (`is`, `is not`)

Identity operators are used to compare the objects, **not if they are equal**, but if they are actually the **same object**, with the same memory location.

- `is`: Returns True if both variables are the same object.
- `is not`: Returns True if both variables are not the same object.

**Input:**
```python
x = ["apple", "banana"]
y = ["apple", "banana"]
z = x

# Returns True because z is the same object as x
print(x is z)

# Returns False because x is not the same object as y, even if they have the same content
print(x is y)

# To demonstrate the difference between "is" and "==":
# This returns True because x is equal to y
print(x == y)
```
**Output:**
```text
True
False
True
```

---

## 2. Membership Operators (`in`, `not in`)

Membership operators are used to test if a sequence (such as a string, list, a tuple, or a dictionary) is presented in an object.

- `in`: Returns True if a sequence with the specified value is present in the object.
- `not in`: Returns True if a sequence with the specified value is not present in the object.

**Input:**
```python
fruits = ["apple", "banana"]
name = "Alexander"

# Returns True because "banana" is in the list
print("banana" in fruits)

# Returns True because "pineapple" is NOT in the list
print("pineapple" not in fruits)

# It also works wonderfully on strings!
print("Alex" in name)
print("z" in name)
```
**Output:**
```text
True
True
True
False
```

## 3. Pythonic Deep Dive: Under-the-Hood Behaviors

### A. The Integer Caching Gotcha (`is` vs `==`)
In Python, integers in the range of `-5` to `256` are pre-allocated and cached in memory when the Python interpreter starts. This means any variables holding these numbers will point to the exact same memory address! 

However, numbers outside this range are usually allocated dynamically on separate memory addresses.

**Input:**
```python
# Numbers within the cached range (-5 to 256)
a = 256
b = 256
print("a == b (Values):", a == b)
print("a is b (Identity):", a is b)  # True because of integer caching!

# Numbers outside the cached range
x = 257
y = 257
print("x == y (Values):", x == y)
print("x is y (Identity):", x is y)  # False! They are separate objects in memory.
```
**Output (in interactive REPL):**
```text
a == b (Values): True
a is b (Identity): True
x == y (Values): True
x is y (Identity): False
```

> [!NOTE]
> **REPL vs. Script Compilation Behavior:** If you run `x = 257` and `y = 257` line-by-line in the interactive terminal (REPL), `x is y` will output `False`. However, if you run them inside a script file, Python's compiler compiles the entire function or module together and optimizes identical literals to refer to the exact same object in memory (a process called constant folding), which might cause `x is y` to print `True`. Despite this compiler optimization, the core rule remains: only integers between `-5` and `256` are guaranteed to be cached at runtime.

> [!WARNING]
> Always use `==` for comparing values (numbers, strings, lists). Only use `is` when checking object identity, such as comparing a variable to `None` (e.g., `if val is None:`).

### B. Membership Testing in Dictionaries (Keys vs. Values)
When you use the `in` or `not in` operator on a dictionary, Python checks the **keys** of the dictionary by default, not the values!

Checking membership in a dictionary's keys is extremely fast (takes $O(1)$ constant time) because dictionaries use hash tables under the hood. Checking list membership takes $O(n)$ linear time.

**Input:**
```python
# A simple dictionary
student_scores = {"Alice": 95, "Bob": 88}

# Checks keys by default
print("Is 'Alice' in student_scores?", "Alice" in student_scores)
print("Is 95 in student_scores?", 95 in student_scores)  # False! 95 is a value, not a key.

# To check values, you must explicitly look in .values()
print("Is 95 in student_scores.values()?", 95 in student_scores.values())
```
**Output:**
```text
Is 'Alice' in student_scores? True
Is 95 in student_scores? False
Is 95 in student_scores.values()? True
```
