# 1. Immutability and Unpacking

Tuples are another built-in data type used to store collections of data. They are very similar to Lists, but with one massive, fundamental difference: **Tuples are Immutable (Unchangeable).**

---

## 1. Tuple Characteristics
Tuples are written with round brackets `()`.
- **Ordered:** The items have a defined order, and that order will not change.
- **Unchangeable (Immutable):** Once a tuple is created, you absolutely cannot add, remove, or change any items.
- **Allow Duplicates:** Tuples are indexed, so they can have items with the same value.

**Input:**
```python
mytuple = ("apple", "banana", "cherry", "apple")
print(mytuple)
print("Length:", len(mytuple))
```
**Output:**
```text
('apple', 'banana', 'cherry', 'apple')
Length: 4
```

### Creating a Tuple with One Item
To create a tuple with only one item, you **must add a comma** after the item. If you do not add a comma, Python will not recognize it as a tuple; it will just think it's a string surrounded by math parentheses!

**Input:**
```python
thistuple = ("apple",)
print(type(thistuple))

# NOT a tuple
not_a_tuple = ("apple")
print(type(not_a_tuple))
```
**Output:**
```text
<class 'tuple'>
<class 'str'>
```

---

## 2. Accessing Tuple Items
You can access tuple items by referring to the index number, exactly like a List. Slicing and negative indexing work exactly the same way too!

**Input:**
```python
thistuple = ("apple", "banana", "cherry", "orange", "kiwi", "melon")

# Access the second item
print(thistuple[1])

# Slice the tuple
print(thistuple[2:5])
```
**Output:**
```text
banana
('cherry', 'orange', 'kiwi')
```

---

## 3. The "Workaround" for Updating Tuples
Because tuples are immutable, you cannot do `mytuple[0] = "kiwi"` or `mytuple.append("orange")`. Python will crash with a `TypeError`.

**So how do you update a tuple?** 
The secret is to convert the tuple into a List, make your changes to the List, and then convert the List back into a tuple!

**Input:**
```python
x = ("apple", "banana", "cherry")

# 1. Convert to list
y = list(x)

# 2. Make the changes (lists are mutable!)
y[1] = "kiwi"
y.append("orange")

# 3. Convert back to tuple
x = tuple(y)

print(x)
```
**Output:**
```text
('apple', 'kiwi', 'cherry', 'orange')
```

---

## 4. Unpacking a Tuple
When we create a tuple, we normally assign values to it. This is called "packing" a tuple.
Python also allows us to extract the values back into variables. This is called **"unpacking"**.

**Input:**
```python
# Packing a tuple
fruits = ("apple", "banana", "cherry")

# Unpacking a tuple into variables
(green, yellow, red) = fruits

print(green)
print(yellow)
print(red)
```
**Output:**
```text
apple
banana
cherry
```

### Using Asterisk `*` (Rest Operator)
If the number of variables is less than the number of values, you can add an `*` to the variable name. That variable will automatically become a List containing the remaining values!

**Input:**
```python
fruits = ("apple", "banana", "cherry", "strawberry", "raspberry")

(green, yellow, *reds) = fruits

print("Green:", green)
print("Yellow:", yellow)
print("All the Reds:", reds) # Notice this prints as a List!
```
**Output:**
```text
Green: apple
Yellow: banana
All the Reds: ['cherry', 'strawberry', 'raspberry']
```

## 5. Simple Built-in Functions (`any` and `all`)
You can quickly verify the contents of a tuple without writing a loop.
- `all()` returns `True` if every single item is "truthy" (not False, not 0, and not empty).
- `any()` returns `True` if at least one item is "truthy".

**Input:**
```python
boolean_tuple = (False, False, True, False)

print("Are all true?", all(boolean_tuple))
print("Are any true?", any(boolean_tuple))
```
**Output:**
```text
Are all true? False
Are any true? True
```
