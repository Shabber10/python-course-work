# 2. Classic Recursion Problems

Solving classic algorithm problems is the best way to understand how recursion simplifies complex logic.

---

## 1. Factorial of a Number
The factorial of a non-negative integer $n$ (written as $n!$) is the product of all positive integers less than or equal to $n$.

### Mathematical Definition:
- $n! = n \times (n-1)!$
- **Base Case:** $0! = 1$ and $1! = 1$

### Python Implementation:
**Input:**
```python
def factorial(n):
    # 1. Base Case
    if n == 0 or n == 1:
        return 1
    # 2. Recursive Case
    return n * factorial(n - 1)

print("Factorial of 5:", factorial(5))
```
**Output:**
```text
Factorial of 5: 120
```

## 2. Fibonacci Sequence
The Fibonacci sequence is a series of numbers where each number is the sum of the two preceding ones, starting from 0 and 1.
`0, 1, 1, 2, 3, 5, 8, 13, 21, ...`

### Mathematical Definition:
- $F(n) = F(n-1) + F(n-2)$
- **Base Cases:** $F(0) = 0$, $F(1) = 1$

### Python Implementation:
**Input:**
```python
def fibonacci(n):
    # 1. Base Cases
    if n == 0:
        return 0
    if n == 1:
        return 1
    # 2. Recursive Case
    return fibonacci(n - 1) + fibonacci(n - 2)

# Get the 6th Fibonacci number (0-indexed: 0, 1, 2, 3, 4, 5, 6)
print("6th Fibonacci number is:", fibonacci(6))
```
**Output:**
```text
6th Fibonacci number is: 8
```

### 🌳 Recursion Tree for `fibonacci(4)`
To understand how execution branches out, let's look at the recursion tree for calculating the 4th Fibonacci number:

```text
                  fib(4)
                /        \
            fib(3)        fib(2)
           /      \       /      \
       fib(2)   fib(1)  fib(1)  fib(0)
      /      \
  fib(1)   fib(0)
```

> [!TIP]
> Notice how `fib(2)` is calculated **twice**, and `fib(1)` is calculated **three times**! Naive recursive Fibonacci has an exponential time complexity of $O(2^n)$. To optimize it, we use **Memoization** (storing results of functions in a dictionary) to avoid recalculating them.

## 3. Sum of Digits
Given an integer $n$, find the sum of all its digits recursively.
- Extract the last digit: `n % 10`
- Remove the last digit: `n // 10`

### Python Implementation:
**Input:**
```python
def sum_of_digits(n):
    # Base Case: single digit number
    if n < 10:
        return n
    # Recursive Case
    return (n % 10) + sum_of_digits(n // 10)

print("Sum of digits of 12345:", sum_of_digits(12345))
```
**Output:**
```text
Sum of digits of 12345: 15
```

## 4. Recursion vs. Iteration
| Parameter | Recursion | Iteration |
| :--- | :--- | :--- |
| **Concept** | Function calls itself. | Repeatedly executes a loop block. |
| **Code Size** | Usually shorter, cleaner, and more intuitive. | Can be longer or more complex to read. |
| **Memory** | High (each call adds a frame to stack memory). | Low (uses constant stack space). |
| **Performance** | Can be slower due to overhead of call stack. | Faster and more efficient. |
| **Termination** | Reached via a **Base Case**. | Reached via a **Loop Condition** failure or break. |
