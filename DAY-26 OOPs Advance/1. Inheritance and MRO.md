# 1. Inheritance and MRO

Inheritance is one of the foundational pillars of Object-Oriented Programming. It allows you to create a new class (subclass or child class) that inherits the attributes and methods of an existing class (base class or parent class). This promotes code reuse and structures related concepts in a logical hierarchy.

---

## 1. The Concept of Inheritance

Why write the same variables and methods over and over? If we have a smart home system with smart lights, smart thermostats, and smart speakers, they all share basic characteristics: a name, a power status, and controls to turn them on or off. 

Instead of writing `turn_on()` for every device class, we write it once in a base `SmartDevice` class and let the others inherit it.

---

## 1.5. Is-A vs. Has-A Relationships (Composition vs. Inheritance)

When designing classes, developers must choose the correct relationship between objects:

1. **Is-A Relationship (Inheritance):** A subclass is a more specific version of its parent class.
   - *Example:* A `SmartLight` **is a** `SmartDevice`. A `Dog` **is an** `Animal`.
2. **Has-A Relationship (Composition / Aggregation):** A class contains one or more instances of another class as parts of its data.
   - *Example:* A `Department` **has a** list of `Courses`. A `Car` **has an** `Engine`.

Here is a quick example of a **Has-A** relationship (Composition) in action:

**Input:**
```python
class Course:
    def __init__(self, course_name):
        self.course_name = course_name

class Department:
    def __init__(self, dept_name):
        self.dept_name = dept_name
        self.courses = []  # Department HAS-A list of Courses

    def add_course(self, course):
        self.courses.append(course)

    def display_courses(self):
        print(f"Courses in {self.dept_name}:")
        for course in self.courses:
            print(f"- {course.course_name}")

dept = Department("Computer Science")
dept.add_course(Course("Python Programming"))
dept.add_course(Course("Data Structures"))
dept.display_courses()
```
**Output:**
```text
Courses in Computer Science:
- Python Programming
- Data Structures
```

### A. Single Inheritance
A child class inherits from a single parent class.

**Input:**
```python
# Parent Class
class SmartDevice:
    def __init__(self, name):
        self.name = name
        self.is_on = False

    def turn_on(self):
        self.is_on = True
        print(f"рҹ”Ң {self.name} is now turned ON.")

# Child Class
class SmartLight(SmartDevice):
    def set_brightness(self, level):
        print(f"рҹ’Ў {self.name} brightness set to {level}%.")

light = SmartLight("Living Room Light")
light.turn_on()  # Inherited method
light.set_brightness(75)  # Child-specific method
```
**Output:**
```text
рҹ”Ң Living Room Light is now turned ON.
рҹ’Ў Living Room Light brightness set to 75%.
```

### B. Multilevel Inheritance
A child class inherits from a parent class, which in turn inherits from another grandparent class.

**Input:**
```python
class Device:
    def __init__(self, brand):
        self.brand = brand

class SmartDevice(Device):
    def connect_to_wifi(self):
        print(f"рҹ“¶ Connected device to smart hub.")

class SmartSpeaker(SmartDevice):
    def play_music(self):
        print(f"рҹҺө playing music from {self.brand} smart speaker.")

speaker = SmartSpeaker("Sonos")
print("Brand:", speaker.brand)  # Inherited from Device
speaker.connect_to_wifi()       # Inherited from SmartDevice
speaker.play_music()            # Child class method
```
**Output:**
```text
Brand: Sonos
рҹ“¶ Connected device to smart hub.
рҹҺө playing music from Sonos smart speaker.
```

### C. Hierarchical Inheritance
Multiple child classes inherit from a single parent class.

**Input:**
```python
class SmartDevice:
    def reboot(self):
        print("рҹҢҖ Rebooting device software...")

class SmartPlug(SmartDevice):
    def toggle_power(self):
        print("вҡЎ Toggling outlet relays.")

class SmartCamera(SmartDevice):
    def start_stream(self):
        print("рҹҺҘ Streaming live feed to cloud.")

plug = SmartPlug()
camera = SmartCamera()

plug.reboot()  # Shared inherited method
camera.reboot()  # Shared inherited method
```
**Output:**
```text
рҹҢҖ Rebooting device software...
рҹҢҖ Rebooting device software...
```

> [!WARNING]
> **Sibling Class Access Boundaries (Crucial OOP Rule):**
> In hierarchical inheritance, peer child classes (like `SmartPlug` and `SmartCamera`) inherit common traits from the parent, but they are **independent siblings**. A `SmartCamera` object has absolutely **no access** to methods unique to `SmartPlug` (like `toggle_power()`). Attempting to call a sibling's unique method will instantly raise an `AttributeError`!

### D. Multiple Inheritance
A child class inherits attributes and methods from more than one parent class.

**Input:**
```python
class Camera:
    def take_photo(self):
        print("рҹ“ё Capturing high-resolution photo.")

class Phone:
    def make_call(self, number):
        print(f"рҹ“һ Dialing {number}...")

# Smartphone inherits from BOTH parents!
class Smartphone(Camera, Phone):
    def browse_web(self):
        print("рҹҢҗ Opening web browser...")

phone = Smartphone()
phone.take_photo()  # From Camera parent
phone.make_call("9876543210")  # From Phone parent
phone.browse_web()  # From child
```
**Output:**
```text
рҹ“ё Capturing high-resolution photo.
рҹ“һ Dialing 9876543210...
рҹҢҗ Opening web browser...
```

### E. Hybrid Inheritance
A combination of two or more types of inheritance. A classic example is the **Diamond Problem** (visualized in the diagram above).

#### Real-World Case Study: TeamLead
In corporate software systems, inheritance models can quickly become hybrid. For instance:
* `Person` is a base class.
* `Employee` inherits from `Person` (**Multilevel Inheritance**).
* `Project` is an independent base class.
* `TeamLead` inherits from both `Employee` and `Project` (**Multiple Inheritance**).

This combined architecture is **Hybrid Inheritance**.

**Input:**
```python
class Person:
    def __init__(self, name):
        self.name = name

class Employee(Person):
    def get_role(self):
        return "Employee"

class Project:
    def __init__(self, project_name):
        self.project_name = project_name

# TeamLead represents Hybrid Inheritance
class TeamLead(Employee, Project):
    def __init__(self, name, project_name):
        Employee.__init__(self, name)
        Project.__init__(self, project_name)

    def show_details(self):
        print(f"Lead: {self.name} | Project: {self.project_name} | Role: {self.get_role()}")

lead = TeamLead("Arjun", "Smart Home Automation")
lead.show_details()
```
**Output:**
```text
Lead: Arjun | Project: Smart Home Automation | Role: Employee
```

## 3. Constructor Overriding and `super()`

If a child class does not define an `__init__` constructor, it automatically calls the parent's `__init__`. However, if the child class defines its own `__init__` method, it **overrides** the parent's constructor. 

To run the parent's constructor alongside the child's constructor, we use the `super()` function.

**Input:**
```python
class SmartDevice:
    def __init__(self, name, brand):
        self.name = name
        self.brand = brand
        self.is_on = False

class SmartThermostat(SmartDevice):
    def __init__(self, name, brand, temp_setting):
        # Call the parent class __init__ constructor using super()
        super().__init__(name, brand)
        # Initialize child-specific properties
        self.temp_setting = temp_setting

    def display_status(self):
        print(f"Device: {self.name} | Brand: {self.brand} | Target Temp: {self.temp_setting}В°C")

thermostat = SmartThermostat("Living Room Thermostat", "Nest", 22)
thermostat.display_status()
```
**Output:**
```text
Device: Living Room Thermostat | Brand: Nest | Target Temp: 22В°C
```

### Explicit Constructor Calls in Multiple Inheritance
While `super()` is the standard way to delegate to parent classes, in complex multiple inheritance scenarios where parent classes require different signatures (arguments), calling `super()` once might fail or resolve ambiguously. 

In these cases, we initialize parent constructors explicitly by referencing the class name directly and passing `self` manually.

**Input:**
```python
class Person:
    def __init__(self, name):
        self.name = name

class Job:
    def __init__(self, salary):
        self.salary = salary

# Child inherits from Person and Job (different constructor arguments!)
class Employee(Person, Job):
    def __init__(self, name, salary, role):
        # Explicit parent constructor initialization
        Person.__init__(self, name)
        Job.__init__(self, salary)
        self.role = role

    def show_info(self):
        print(f"Name: {self.name} | Salary: ${self.salary} | Role: {self.role}")

emp = Employee("Alice", 85000, "Developer")
emp.show_info()
```
**Output:**
```text
Name: Alice | Salary: $85000 | Role: Developer
```

> [!NOTE]
> **Why `super()` is preferred when possible:**
> `super()` is dynamic: it returns a proxy object that delegates method calls to a parent or sibling class based on the MRO. This ensures each class in a diamond hierarchy is initialized exactly once, avoiding duplicate constructor calls which can occur when using explicit `Parent.__init__(self)` calls.

## 4. Method Resolution Order (MRO)

When using multiple inheritance, what happens if two parent classes define the exact same method? Which parent's method does the child run?

Python resolves this conflict using a deterministic order called **Method Resolution Order (MRO)**, calculated using the **C3 Linearization** algorithm.

### The Diamond Problem
Imagine a class hierarchy forming a diamond shape:

```text
      в”Ңв”Җв”Җв”Җв”җ
      в”Ӯ A в”Ӯ (defines greet())
     в”Ңв”ҙв”Җв”Җв”Җв”ҙв”җ
     в–ј     в–ј
   в”Ңв”Җв”Җв”Җв”җ в”Ңв”Җв”Җв”Җв”җ
   в”Ӯ B в”Ӯ в”Ӯ C в”Ӯ (both override greet())
   в””в”Җв”¬в”Җв”ҙв”Җв”¬в”Җв”ҳ
     в–ј   в–ј
     в”Ңв”Җв”Җв”Җв”җ
     в”Ӯ D в”Ӯ (inherits from B and C)
     в””в”Җв”Җв”Җв”ҳ
```

If we call `D().greet()`, which `greet()` gets executed?

**Input:**
```python
class A:
    def greet(self):
        print("Greeting from A")

class B(A):
    def greet(self):
        print("Greeting from B")

class C(A):
    def greet(self):
        print("Greeting from C")

class D(B, C):
    pass

d_instance = D()
d_instance.greet()
```
**Output:**
```text
Greeting from B
```

### Inspecting the MRO
You can inspect the exact lookup hierarchy using the `__mro__` attribute or the `mro()` method on the class.

**Input:**
```python
# print MRO list
print(D.mro())
```
**Output:**
```text
[<class '__main__.D'>, <class '__main__.B'>, <class '__main__.C'>, <class '__main__.A'>, <class 'object'>]
```

### How Python's MRO Rules Work:
1. **Depth First, then Breadth (almost):** Look in the class itself first, then left-to-right through its base classes.
2. **Subclasses before Parents:** A parent class is visited only after all its child classes have been checked (which is why `A` is visited last in the diamond problem).
3. **Preserve order:** Python maintains the left-to-right declaration order of base classes in your definition (e.g., `class D(B, C)` searches `B` before `C`).

## 5. Built-in Inheritance Helpers: `isinstance()` and `issubclass()`

Python provides two built-in functions to programmatically inspect inheritance relationships at runtime:

- **`isinstance(object, class)`:** Checks if an **object (instance)** belongs to a specific class, or to any class in its inheritance tree (subclasses).
- **`issubclass(child_class, parent_class)`:** Checks if one **class** inherits directly or indirectly from another class.

**Input:**
```python
class Animal:
    pass

class Dog(Animal):
    pass

# Create an instance
buddy = Dog()

# 1. Using isinstance() - Checks if object belongs to class/subclass
print("Is buddy a Dog?    :", isinstance(buddy, Dog))
print("Is buddy an Animal?:", isinstance(buddy, Animal))
print("Is buddy a string? :", isinstance(buddy, str))

# 2. Using issubclass() - Checks if class inherits from another class
print("\nIs Dog a subclass of Animal?    :", issubclass(Dog, Animal))
print("Is Animal a subclass of Dog?    :", issubclass(Animal, Dog))
print("Is Dog a subclass of object?    :", issubclass(Dog, object))  # All classes inherit from object!
```
**Output:**
```text
Is buddy a Dog?    : True
Is buddy an Animal?: True
Is buddy a string? : False

Is Dog a subclass of Animal?    : True
Is Animal a subclass of Dog?    : False
Is Dog a subclass of object?    : True
```
