# 3. Polymorphism and Duck Typing

Polymorphism means "many forms." In programming, it refers to the ability of different classes to share the same method name but execute different actions depending on the object that calls them.

---

## 1. Method Overriding

Method overriding occurs when a child class provides a specific implementation of a method that is already defined in its parent class. This allows the child class to tailor inherited behaviors.

Let's build a payment gateway processor system.

**Input:**
```python
# Base Gateway
class PaymentGateway:
    def process_payment(self, amount):
        print(f"Processing a generic payment of ${amount}...")

# Child Gateways override the base payment method
class StripePayment(PaymentGateway):
    def process_payment(self, amount):
        print(f"💳 charging credit card via Stripe API for ${amount} (fee: $0.30).")

class PayPalPayment(PaymentGateway):
    def process_payment(self, amount):
        print(f"💰 Routing PayPal express payment of ${amount} (fee: 2.9%).")

# Polymorphic Function: takes any gateway and processes payment
def checkout(gateway, amount):
    gateway.process_payment(amount)

# Test polymorphism
checkout(StripePayment(), 100)
checkout(PayPalPayment(), 100)
```
**Output:**
```text
💳 charging credit card via Stripe API for $100 (fee: $0.30).
💰 Routing PayPal express payment of $100 (fee: 2.9%).
```

## 2. Method Overloading (and Pythonic Workarounds)

In languages like Java or C++, you can define multiple methods with the *same name* but *different parameter signatures* (e.g. `register(email)` and `register(email, phone)`). This is called **Method Overloading**.

### ⚠️ Python is different:
Because Python is dynamically typed and interprets method arguments flexibly, defining two methods with the same name will cause the second definition to **overwrite** the first one!

**Input:**
```python
class Registry:
    def register(self, email):
        print("Registered via Email:", email)

    def register(self, email, phone):  # Overwrites the first register method!
        print(f"Registered via Email: {email} | Phone: {phone}")

reg = Registry()
try:
    reg.register("user@test.com")
except TypeError as e:
    print("Error:", e)
```
**Output:**
```text
Error: Registry.register() missing 1 required positional argument: 'phone'
```

### Pythonic Solutions for Overloading:
To achieve the behavior of method overloading in Python, we use these common strategies:

#### 1. Default Arguments (Recommended)
Set optional arguments to `None` or a default value inside the signature.

```python
class Registry:
    def register(self, email, phone=None):
        if phone is None:
            print("Registered via Email:", email)
        else:
            print(f"Registered via Email: {email} | Phone: {phone}")
```

#### 2. Variable Arguments (`*args` or `**kwargs`)
Accept a limitless configuration of incoming arguments.

```python
class Calculator:
    def add(self, *args):
        return sum(args)
```

#### 3. Using `@overload` from the `typing` module
For static analysis and IDE autocompletion, you can use the `@overload` decorator, followed by one final concrete method.

```python
from typing import overload

class UserPortal:
    @overload
    def fetch_user(self, user_id: int) -> str: ...
    
    @overload
    def fetch_user(self, email: str) -> str: ...

    # The actual single concrete method that runs at runtime
    def fetch_user(self, identifier):
        if isinstance(identifier, int):
            return f"User info for ID: {identifier}"
        return f"User info for Email: {identifier}"
```

## 3. Operator Overloading

Operator overloading allows you to define how built-in operators (like `+`, `-`, `*`, `==`, `<`) behave when applied to your custom objects.

Python achieves this by mapping operators to special dunder magic methods:

| Operator | Magic Method |
| :---: | :--- |
| `+` | `__add__(self, other)` |
| `-` | `__sub__(self, other)` |
| `*` | `__mul__(self, other)` |
| `==` | `__eq__(self, other)` |
| `<` | `__lt__(self, other)` |
| `len()` | `__len__(self)` |

Let's build a shopping cart `Item` class and overload `+` to add items together, and `==` to compare items.

**Input:**
```python
class Item:
    def __init__(self, name, price):
        self.name = name
        self.price = price

    # Overload the addition '+' operator
    def __add__(self, other):
        if isinstance(other, Item):
            return self.price + other.price
        raise TypeError("Can only add two Item instances together!")

    # Overload the equality '==' operator
    def __eq__(self, other):
        if isinstance(other, Item):
            return self.name == other.name and self.price == other.price
        return False

# Create items
item1 = Item("Keyboard", 80)
item2 = Item("Mouse", 40)
item3 = Item("Keyboard", 80)

# Add items using '+'
total_price = item1 + item2
print("Total Price:", total_price)

# Compare items using '=='
print("Is item1 equal to item2?", item1 == item2)
print("Is item1 equal to item3?", item1 == item3)
```
**Output:**
```text
Total Price: 120
Is item1 equal to item2? False
Is item1 equal to item3? True
```

### Overloading the Multiplication `*` Operator (`__mul__`)
You can also overload the multiplication `*` operator by defining the **`__mul__(self, other)`** method. This allows custom scaling or replication operations on class instances.

**Input:**
```python
class RepeatString:
    def __init__(self, text):
        self.text = text

    # Overload the '*' operator
    def __mul__(self, multiplier):
        if isinstance(multiplier, int):
            return self.text * multiplier
        raise TypeError("Multiplier must be an integer!")

# Test the overloaded * operator
repeat_obj = RepeatString("Hello")
print("Result of multiplication:", repeat_obj * 3)
```
**Output:**
```text
Result of multiplication: HelloHelloHello
```

## 4. Duck Typing

Duck Typing is a core design philosophy in Python's dynamic typing system. It gets its name from the saying: 
> *"If it walks like a duck and quacks like a duck, then it must be a duck."*

In statically-typed languages, a function might require a parameter to belong to a specific class hierarchy (e.g. `def play(device: AudioDevice)`). 

In Python, the function simply calls a method (e.g., `device.play()`). Python doesn't care about the object's class inheritance tree; it only cares that **the object supports the requested behavior (method)**.

Let's see this in action:

**Input:**
```python
class CDPlayer:
    def play(self):
        print("💿 Spinning CD and playing audio tracks.")

class MP3Player:
    def play(self):
        print("📱 Decoding MP3 file and outputting digital audio.")

class Radio:
    def play(self):
        print("📻 Tuning receiver frequency and streaming FM broadcast.")

# Duck Typed function: cares only that 'device' has a '.play()' method
def start_music(device):
    device.play()

# Test with unrelated classes that share behavior
devices = [CDPlayer(), MP3Player(), Radio()]
for dev in devices:
    start_music(dev)
```
**Output:**
```text
💿 Spinning CD and playing audio tracks.
📱 Decoding MP3 file and outputting digital audio.
📻 Tuning receiver frequency and streaming FM broadcast.
```
