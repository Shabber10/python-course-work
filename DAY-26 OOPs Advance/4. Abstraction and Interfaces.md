# 4. Abstraction and Interfaces

Abstraction is the process of hiding internal implementation details and showing only the essential features of an object. It provides a clean, simplified blueprint that ensures other developers adhere to a consistent architecture.

---

## 1. Abstract Base Classes (ABC)

In Python, we cannot define abstract classes natively. Instead, we use the built-in `abc` module. An abstract class inherits from `ABC` (Abstract Base Class) and defines one or more **abstract methods** using the `@abstractmethod` decorator.

### Key Rules:
- **Cannot Instantiate:** You cannot create an object directly from an abstract class.
- **Strict Enforcement:** Any subclass inheriting from an abstract class **must** implement all of its abstract methods. If it fails to do so, Python will raise a `TypeError` when you try to instantiate the subclass.

Let's build a media player control system blueprint.

**Input:**
```python
from abc import ABC, abstractmethod

# Abstract Base Class (Blueprint)
class MediaPlayer(ABC):
    def __init__(self, filename):
        self.filename = filename

    # Abstract Method: Subclasses MUST implement this
    @abstractmethod
    def play(self):
        pass

    # Abstract Method: Subclasses MUST implement this
    @abstractmethod
    def pause(self):
        pass

    # Concrete Method: Inherited as-is by subclasses
    def display_metadata(self):
        print(f"🎬 File: {self.filename}")

# 1. Trying to instantiate the abstract class directly will fail!
try:
    player = MediaPlayer("song.mp3")
except TypeError as e:
    print("Instantiation Error:", e)
```
**Output:**
```text
Instantiation Error: Can't instantiate abstract class MediaPlayer with abstract methods pause, play
```

---

## 2. Implementing the Blueprint

Let's write a subclass that implements these methods.

**Input:**
```python
class AudioPlayer(MediaPlayer):
    def play(self):
        print(f"🔊 Playing audio file: {self.filename}")

    def pause(self):
        print("⏸️ Audio playback paused.")

# If we implement all abstract methods, instantiation is successful!
mp3 = AudioPlayer("podcast.mp3")
mp3.display_metadata()  # Concrete parent method
mp3.play()              # Overridden abstract method
mp3.pause()             # Overridden abstract method
```
**Output:**
```text
🎬 File: podcast.mp3
🔊 Playing audio file: podcast.mp3
⏸️ Audio playback paused.
```

---

## 3. What Happens If You Forget a Method?

If a subclass implements `play()` but forgets to implement `pause()`, Python will treat the subclass as abstract as well and block its instantiation.

**Input:**
```python
class IncompletePlayer(MediaPlayer):
    def play(self):
        print(f"🔊 Playing: {self.filename}")
    
    # pause() is missing!

try:
    bad_player = IncompletePlayer("video.mp4")
except TypeError as e:
    print("Subclass Error:", e)
```
**Output:**
```text
Subclass Error: Can't instantiate abstract class IncompletePlayer with abstract method pause
```

## 4. Interfaces in Python

In languages like Java, C#, or Go, an **Interface** is a formal contract containing only method signatures without any state or implementations.

Python does not have a separate `interface` keyword. Instead, we define an interface by creating an Abstract Base Class that contains **only** abstract methods and no concrete methods or instance attributes.

Let's build a database connector interface that forces different database adapters to support the same operations.

**Input:**
```python
from abc import ABC, abstractmethod

# Interface: 100% abstract methods, no concrete states
class DatabaseConnector(ABC):
    @abstractmethod
    def connect(self):
        pass

    @abstractmethod
    def disconnect(self):
        pass

    @abstractmethod
    def execute_query(self, query):
        pass

# Implementing the Interface for MySQL
class MySQLConnector(DatabaseConnector):
    def connect(self):
        print("🔌 Connecting to MySQL database on port 3306...")

    def disconnect(self):
        print("🔌 Disconnecting from MySQL server.")

    def execute_query(self, query):
        print(f"⚡ MySQL executing query: {query}")

# Implementing the Interface for MongoDB
class MongoDBConnector(DatabaseConnector):
    def connect(self):
        print("🍃 Connecting to MongoDB Atlas cluster on port 27017...")

    def disconnect(self):
        print("🍃 Disconnecting from MongoDB database connection.")

    def execute_query(self, query):
        print(f"⚡ MongoDB running aggregation: {query}")

# Polymorphic database manager
def run_backup(connector):
    connector.connect()
    connector.execute_query("SELECT * FROM users")
    connector.disconnect()

# Test database adapters
mysql_adapter = MySQLConnector()
mongo_adapter = MongoDBConnector()

print("--- Running MySQL Backup ---")
run_backup(mysql_adapter)

print("\n--- Running MongoDB Backup ---")
run_backup(mongo_adapter)
```
**Output:**
```text
--- Running MySQL Backup ---
🔌 Connecting to MySQL database on port 3306...
⚡ MySQL executing query: SELECT * FROM users
🔌 Disconnecting from MySQL server.

--- Running MongoDB Backup ---
🍃 Connecting to MongoDB Atlas cluster on port 27017...
⚡ MongoDB running aggregation: SELECT * FROM users
🍃 Disconnecting from MongoDB database connection.
```

## 5. Summary: Abstraction vs. Encapsulation

Students often confuse Abstraction and Encapsulation because they both deal with "hiding" things. Here is a clear way to distinguish them:

| Concept | What It Hides | Primary Goal | Implementation |
| :--- | :--- | :--- | :--- |
| **Abstraction** | Hides *complexity* of implementation. | Shows *what* an object does (high-level blueprint). | Abstract base classes (`ABC`), interfaces. |
| **Encapsulation** | Hides *data* and direct state access. | Shows *how* to safely read/write internal states. | Access modifiers (`_`, `__`), getters/setters (`@property`). |
