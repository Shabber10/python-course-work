# 2. Generator Expressions and Performance

Just as Python supports list comprehensions, it also supports **generator expressions**. Generator expressions allow you to create generators quickly using a single line of code.

---

## 1. Syntax of Generator Expressions
The syntax of a generator expression is identical to list comprehensions, except that it uses parentheses `()` instead of square brackets `[]`.

| Type | Syntax | Return Type | Memory Behavior |
| :--- | :--- | :--- | :--- |
| **List Comprehension** | `[x*x for x in range(n)]` | List | Generates all items immediately. |
| **Generator Expression** | `(x*x for x in range(n))` | Generator | Generates items one by one on demand. |

**Input:**
```python
# List Comprehension
list_comp = [x ** 2 for x in range(5)]
print("List Comprehension:", list_comp)

# Generator Expression
gen_exp = (x ** 2 for x in range(5))
print("Generator Expression Object:", gen_exp)

# Reading from the generator expression
for val in gen_exp:
    print(val, end=" ")
print()
```
**Output:**
```text
List Comprehension: [0, 1, 4, 9, 16]
Generator Expression Object: <generator object <genexpr> at 0x...>
0 1 4 9 16 
```

---

## 2. Memory Performance Comparison
To see the true power of generators, let us compare the memory size of a list versus a generator containing 1 million elements.

We use the `sys.getsizeof()` function to measure the memory consumption in bytes.

**Input:**
```python
import sys

# List of 1 million numbers
huge_list = [i for i in range(1000000)]
print("Memory used by List:", sys.getsizeof(huge_list), "bytes")

# Generator of 1 million numbers
huge_gen = (i for i in range(1000000))
print("Memory used by Generator:", sys.getsizeof(huge_gen), "bytes")
```
**Output:**
```text
Memory used by List: 8448728 bytes
Memory used by Generator: 104 bytes
```

> [!IMPORTANT]
> The list occupies around **8.4 Megabytes** in RAM because all 1 million integers are created and stored at once. 
> The generator occupies only **104 bytes**! It stores only the execution state and generator logic, yielding numbers one by one as they are requested.

---

## 3. Creating Infinite Sequences
Because generators are lazily evaluated, they can represent **infinite sequences** without running out of memory.

**Input:**
```python
def infinite_integers():
    n = 1
    while True:
        yield n
        n += 1

gen = infinite_integers()

# We can retrieve as many numbers as we need, and the program never crashes!
print(next(gen))
print(next(gen))
print(next(gen))
print(next(gen))
```
**Output:**
```text
1
2
3
4
```
