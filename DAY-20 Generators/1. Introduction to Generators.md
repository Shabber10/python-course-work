# 1. Introduction to Generators

A **generator** in Python is a special type of function that returns an iterator object. It allows you to declare a function that behaves like an iterator, producing a sequence of values on the fly instead of computing them all at once and saving them in memory.

---

## 1. The `yield` Keyword
Unlike regular functions that use the `return` keyword to exit and send back a value, a generator function uses the `yield` keyword. 

- `return` terminates the function completely, clearing its local state.
- `yield` pauses the function, saves all of its local states, and returns the yielded value to the caller. When the generator is called again, it resumes execution right after the last `yield` statement.

---

## 2. Defining and Using a Generator
You define a generator just like a regular function, but you use `yield` instead of `return`.

**Input:**
```python
def my_generator():
    print("First item yielded:")
    yield 1
    
    print("Second item yielded:")
    yield 2
    
    print("Third item yielded:")
    yield 3

# Instantiating the generator object
gen = my_generator()
print("Generator object created:", gen)
```
**Output:**
```text
Generator object created: <generator object my_generator at 0x...>
```

### Iterating over the Generator
We can retrieve values from the generator using the built-in `next()` function or a `for` loop.

#### Using `next()`:
**Input:**
```python
print(next(gen))  # Runs until the first yield
print(next(gen))  # Resumes and runs until the second yield
print(next(gen))  # Resumes and runs until the third yield

# Calling next() again will raise StopIteration because there are no more yields!
try:
    print(next(gen))
except StopIteration:
    print("Generator completed, StopIteration exception caught.")
```
**Output:**
```text
First item yielded:
1
Second item yielded:
2
Third item yielded:
3
Generator completed, StopIteration exception caught.
```

#### Using a `for` Loop:
The `for` loop automatically handles the `StopIteration` exception and terminates gracefully.

**Input:**
```python
# A generator function for even numbers up to limit
def even_numbers(limit):
    n = 0
    while n <= limit:
        yield n
        n += 2

for num in even_numbers(6):
    print(num)
```
**Output:**
```text
0
2
4
6
```

---

## 3. Key Benefits of Generators
1. **Lazy Evaluation:** Values are only computed when requested. This saves processor time when working with huge datasets.
2. **State Retention:** Unlike regular functions that start fresh on every invocation, generators remember their local variables and code execution points.
3. **Memory Efficiency:** Since they do not store the entire list of values in memory, they use a fraction of the RAM of list-based sequences.
