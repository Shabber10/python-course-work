# 2. Function Arguments and Return Values

Functions can accept inputs called **arguments** (or parameters) and send back results using a **return statement**.

---

## 1. Parameters vs. Arguments
- **Parameters:** The variables listed in the function definition.
- **Arguments:** The actual values passed to the function when it is called.

```python
def greet(name):  # 'name' is a parameter
    print("Hello", name)

greet("Alice")  # '"Alice"' is an argument
```

---

## 2. Types of Arguments

### A. Positional Arguments
Arguments passed in the correct positional order. The first argument maps to the first parameter, the second to the second, etc.

**Input:**
```python
def describe_pet(animal_type, pet_name):
    print(f"I have a {animal_type} named {pet_name}.")

# Positional call: order matters!
describe_pet("dog", "Buddy")
describe_pet("Buddy", "dog")  # Confusing!
```
**Output:**
```text
I have a dog named Buddy.
I have a Buddy named dog.
```

### B. Keyword Arguments
Arguments identified by the parameter name. Order does not matter!

**Input:**
```python
def describe_pet(animal_type, pet_name):
    print(f"I have a {animal_type} named {pet_name}.")

# Keyword call: order does not matter
describe_pet(pet_name="Buddy", animal_type="dog")
```
**Output:**
```text
I have a dog named Buddy.
```

### C. Default Parameters
Parameters can have default values. If an argument is not provided during call, the default value is used.

> [!WARNING]
> Default parameters must always be placed AFTER non-default parameters in the function definition!

**Input:**
```python
# 'country' has a default value of "Norway"
def my_function(country="Norway"):
    print("I am from " + country)

my_function("Sweden")
my_function()  # Uses default
```
**Output:**
```text
I am from Sweden
I am from Norway
```

---

## 3. Returning Values
The `return` statement exits a function, optionally passing back an expression or value to the caller.

### A. Single Return Value
**Input:**
```python
def square(x):
    return x * x

result = square(4)
print("Square of 4 is:", result)
```
**Output:**
```text
Square of 4 is: 16
```

### B. Returning Multiple Values
You can return multiple values separated by commas. Python bundles them together and returns them as a **tuple**.

**Input:**
```python
def arithmetic_ops(a, b):
    add = a + b
    sub = a - b
    mul = a * b
    return add, sub, mul  # returns a tuple

# Unpacking the returned values
plus, minus, times = arithmetic_ops(10, 5)
print(f"Addition: {plus}, Subtraction: {minus}, Multiplication: {times}")

# Or capture as a single tuple
res = arithmetic_ops(10, 5)
print("Tuple result:", res)
```
**Output:**
```text
Addition: 15, Subtraction: 5, Multiplication: 50
Tuple result: (15, 5, 50)
```

---

## 4. Variable-Length Arguments
Sometimes you do not know beforehand how many arguments will be passed to your function.

### A. Arbitrary Positional Arguments (`*args`)
By prefixing a parameter name with an asterisk (`*`), Python packs all incoming positional arguments into a **tuple**.

**Input:**
```python
def sum_all(*numbers):
    print("Received tuple:", numbers)
    total = 0
    for num in numbers:
        total += num
    return total

print("Sum:", sum_all(1, 2, 3, 4, 5))
```
**Output:**
```text
Received tuple: (1, 2, 3, 4, 5)
Sum: 15
```

### B. Arbitrary Keyword Arguments (`**kwargs`)
By prefixing a parameter name with double asterisks (`**`), Python packs all incoming keyword arguments into a **dictionary**.

**Input:**
```python
def print_user_details(**details):
    print("Received dictionary:", details)
    for key, value in details.items():
        print(f"{key.capitalize()}: {value}")

print_user_details(name="Alice", age=25, city="Paris")
```
**Output:**
```text
Received dictionary: {'name': 'Alice', 'age': 25, 'city': 'Paris'}
Name: Alice
Age: 25
City: Paris
```
