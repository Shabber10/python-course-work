# 2. Function Arguments and Return Values

Functions can accept inputs called **arguments** (or parameters) and send back results using a **return statement**.

---

## 1. Parameters vs. Arguments
- **Parameters:** The variables listed in the function definition.
- **Arguments:** The actual values passed to the function when it is called.

```python
def greet(name):  # 'name' is a parameter
    print("Hello", name)

greet("Alice")  # '"Alice"' is an argument
```

---

## 2. Types of Arguments

### A. Positional Arguments
Arguments passed in the correct positional order. The first argument maps to the first parameter, the second to the second, etc.

**Input:**
```python
def describe_pet(animal_type, pet_name):
    print(f"I have a {animal_type} named {pet_name}.")

# Positional call: order matters!
describe_pet("dog", "Buddy")
describe_pet("Buddy", "dog")  # Confusing!
```
**Output:**
```text
I have a dog named Buddy.
I have a Buddy named dog.
```

### B. Keyword Arguments
Arguments identified by the parameter name. Order does not matter!

**Input:**
```python
def describe_pet(animal_type, pet_name):
    print(f"I have a {animal_type} named {pet_name}.")

# Keyword call: order does not matter
describe_pet(pet_name="Buddy", animal_type="dog")
```
**Output:**
```text
I have a dog named Buddy.
```

### C. Default Parameters
Parameters can have default values. If an argument is not provided during call, the default value is used.

> [!WARNING]
> Default parameters must always be placed AFTER non-default parameters in the function definition!

**Input:**
```python
# 'country' has a default value of "Norway"
def my_function(country="Norway"):
    print("I am from " + country)

my_function("Sweden")
my_function()  # Uses default
```
**Output:**
```text
I am from Sweden
I am from Norway
```

---

## ⚠️ CRITICAL WARNING: Mutable Default Arguments
A very common gotcha in Python is using a **mutable object** (like a list or dictionary) as a default argument.

In Python, default arguments are evaluated **only once** when the function is defined, not every time the function is called. This means that if you modify the default list, it will persist those modifications across subsequent calls!

### The Gotcha Code:
**Input:**
```python
def append_to_list(value, my_list=[]):
    my_list.append(value)
    return my_list

print("Call 1:", append_to_list(10))
print("Call 2:", append_to_list(20))  # Expected [20], but got [10, 20]!
```
**Output:**
```text
Call 1: [10]
Call 2: [10, 20]
```

### The Clean Fix:
Use `None` as the default value and check for it inside the function body.

**Input:**
```python
def append_to_list_safe(value, my_list=None):
    if my_list is None:
        my_list = []  # Creates a fresh list on every execution
    my_list.append(value)
    return my_list

print("Safe Call 1:", append_to_list_safe(10))
print("Safe Call 2:", append_to_list_safe(20))
```
**Output:**
```text
Safe Call 1: [10]
Safe Call 2: [20]
```

---

## 3. Returning Values
The `return` statement exits a function, optionally passing back an expression or value to the caller.

### A. Single Return Value
**Input:**
```python
def square(x):
    return x * x

result = square(4)
print("Square of 4 is:", result)
```
**Output:**
```text
Square of 4 is: 16
```

### B. Returning Multiple Values
You can return multiple values separated by commas. Python bundles them together and returns them as a **tuple**.

**Input:**
```python
def arithmetic_ops(a, b):
    add = a + b
    sub = a - b
    mul = a * b
    return add, sub, mul  # returns a tuple

# Unpacking the returned values
plus, minus, times = arithmetic_ops(10, 5)
print(f"Addition: {plus}, Subtraction: {minus}, Multiplication: {times}")

# Or capture as a single tuple
res = arithmetic_ops(10, 5)
print("Tuple result:", res)
```
**Output:**
```text
Addition: 15, Subtraction: 5, Multiplication: 50
Tuple result: (15, 5, 50)
```

---

## 4. Variable-Length Arguments
Sometimes you do not know beforehand how many arguments will be passed to your function.

### A. Arbitrary Positional Arguments (`*args`)
By prefixing a parameter name with an asterisk (`*`), Python packs all incoming positional arguments into a **tuple**.

**Input:**
```python
def sum_all(*numbers):
    print("Received tuple:", numbers)
    total = 0
    for num in numbers:
        total += num
    return total

print("Sum:", sum_all(1, 2, 3, 4, 5))
```
**Output:**
```text
Received tuple: (1, 2, 3, 4, 5)
Sum: 15
```

### B. Arbitrary Keyword Arguments (`**kwargs`)
By prefixing a parameter name with double asterisks (`**`), Python packs all incoming keyword arguments into a **dictionary**.

**Input:**
```python
def print_user_details(**details):
    print("Received dictionary:", details)
    for key, value in details.items():
        print(f"{key.capitalize()}: {value}")

print_user_details(name="Alice", age=25, city="Paris")
```
**Output:**
```text
Received dictionary: {'name': 'Alice', 'age': 25, 'city': 'Paris'}
Name: Alice
Age: 25
City: Paris
```

## 5. Advanced Parameter Configurations

Python provides precise controls over how arguments can be passed to functions. You can mandate that certain arguments must be positional-only or keyword-only.

### A. Positional-Only Parameters (`/`) - Python 3.8+
By placing a forward slash `/` in the parameters list, all parameters defined *before* the `/` are **positional-only**. They cannot be passed as keyword arguments.

**Input:**
```python
def positional_only(name, /, age):
    print(f"Name: {name}, Age: {age}")

# Valid calls
positional_only("Alice", 25)
positional_only("Alice", age=25)

# Invalid call: passing 'name' as a keyword argument will raise a TypeError
try:
    positional_only(name="Alice", age=25)
except TypeError as e:
    print("Error:", e)
```
**Output:**
```text
Name: Alice, Age: 25
Name: Alice, Age: 25
Error: positional_only() got some positional-only arguments passed as keyword arguments: 'name'
```

### B. Keyword-Only Parameters (`*`)
By placing an asterisk `*` in the parameters list, all parameters defined *after* the `*` are **keyword-only**. They must be passed as keyword arguments.

**Input:**
```python
def keyword_only(name, *, city):
    print(f"Name: {name}, City: {city}")

# Valid call
keyword_only("Bob", city="New York")

# Invalid call: passing 'city' as a positional argument will raise a TypeError
try:
    keyword_only("Bob", "New York")
except TypeError as e:
    print("Error:", e)
```
**Output:**
```text
Name: Bob, City: New York
Error: keyword_only() takes 1 positional argument but 2 were given
```

### C. Argument Unpacking (`*` and `**`)
If you have data stored in a list/tuple or dictionary, you can "unpack" it directly into a function call.
- Use `*` to unpack lists, tuples, or sets into positional arguments.
- Use `**` to unpack dictionaries into keyword arguments.

**Input:**
```python
def display_profile(name, age, city):
    print(f"{name} is {age} years old and lives in {city}.")

# Unpacking a list/tuple
args_list = ["Alice", 25, "Paris"]
display_profile(*args_list)

# Unpacking a dictionary
kwargs_dict = {"name": "Bob", "age": 30, "city": "New York"}
display_profile(**kwargs_dict)
```
**Output:**
```text
Alice is 25 years old and lives in Paris.
Bob is 30 years old and lives in New York.
```
