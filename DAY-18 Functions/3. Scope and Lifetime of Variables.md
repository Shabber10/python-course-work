# 3. Scope and Lifetime of Variables

Not all variables in a Python program are accessible from everywhere. Where a variable is defined determines its **scope** (visibility) and **lifetime** (how long it exists in memory).

---

1. **Local (L):** Names defined inside the current function (using assignment or arguments).
2. **Enclosing (E):** Names defined in the local scope of enclosing functions (outer nested functions).
3. **Global (G):** Names defined at the top-level of the module/script (or declared `global` inside functions).
4. **Built-in (B):** Pre-loaded names in Python (like `len`, `print`, `int`, `ValueError`). This namespace is searched **last** after L, E, and G. If a name is not found here, a `NameError` is raised.

---

## 2. Local Scope
A variable created *inside* a function belongs to the local scope of that function, and can only be used inside that function.

**Input:**
```python
def my_func():
    x = 300  # Local variable
    print("Inside function:", x)

my_func()

# Trying to access x outside the function will raise a NameError!
try:
    print(x)
except NameError as e:
    print("Error:", e)
```
**Output:**
```text
Inside function: 300
Error: name 'x' is not defined
```

---

## 3. Global Scope
A variable created in the main body of the Python code is a **global variable** and belongs to the global scope. Global variables are available from within any scope, global and local.

**Input:**
```python
x = 300  # Global variable

def my_func():
    # We can read global variables directly!
    print("Reading global x inside function:", x)

my_func()
print("Reading global x outside function:", x)
```
**Output:**
```text
Reading global x inside function: 300
Reading global x outside function: 300
```

## 4. Modifying Variables Across Scopes

### A. The `global` Keyword
If you need to **modify** a global variable from inside a function, you must use the `global` keyword. Otherwise, Python will treat it as a new local variable with the same name.

#### Modifying without `global` (Creates a local shadow):
**Input:**
```python
x = 10

def change_val():
    x = 20  # This creates a NEW local variable 'x' and does NOT modify global 'x'
    print("Inside function:", x)

change_val()
print("Outside function:", x)
```
**Output:**
```text
Inside function: 20
Outside function: 10
```

#### Modifying with `global`:
**Input:**
```python
x = 10

def change_val():
    global x  # Declares that we want to use the global 'x'
    x = 20
    print("Inside function (modified global):", x)

change_val()
print("Outside function:", x)
```
**Output:**
```text
Inside function (modified global): 20
Outside function: 20
```

### B. Nested Functions and `nonlocal`
If you define a function inside another function (nested function), the inner function can read variables from the outer function's scope (Enclosing scope). If you want to modify a variable in that enclosing scope, use the `nonlocal` keyword.

**Input:**
```python
def outer_func():
    msg = "hello"
    
    def inner_func():
        nonlocal msg  # Declares that we want to modify the variable in the nearest outer scope
        msg = "hello updated"
        
    inner_func()
    print("Outer function message:", msg)

outer_func()
```
**Output:**
```text
Outer function message: hello updated
```

### C. Namespace Shadowing (Built-in Shadowing Pitfall)
Shadowing happens when you declare a variable or parameter with the same name as a variable or function from an outer namespace (like Python's built-in namespace). This makes the outer name inaccessible inside that scope and can lead to extremely confusing bugs!

**Input:**
```python
# Shadowing a built-in function (len)
def calculate_average(scores):
    # Shadowing the built-in 'len' function by using it as a variable name!
    len_var = 0
    for score in scores:
        len_var += 1
    
    # If we had named 'len_var' as 'len', this call to len(scores) would fail!
    # Let's demonstrate the failure by intentionally shadowing 'sum':
    sum = 100  # Shadowing the built-in sum() function!
    try:
        # This will fail because 'sum' is now an integer, not a function!
        avg = sum(scores) / len_var
    except TypeError as e:
        print("Shadowing Error:", e)

calculate_average([80, 90, 100])
```
**Output:**
```text
Shadowing Error: 'int' object is not callable
```
> [!IMPORTANT]
> Avoid naming variables, arguments, or custom functions after Python built-in functions (such as `list`, `dict`, `str`, `len`, `sum`, `max`, `min`, `type`, etc.).

### D. Closures and Enclosing Scope Realities
A **closure** is a nested function that remembers and has access to variables from its enclosing (outer) function's scope, even after the outer function has finished executing. The outer variables are kept alive in a special area of memory.

**Input:**
```python
def make_multiplier(factor):
    # 'factor' is in the enclosing scope
    def multiply(number):
        return number * factor  # accesses 'factor' from enclosing scope
    return multiply

# Create multiplier functions
double = make_multiplier(2)
triple = make_multiplier(3)

print("Double 5:", double(5))
print("Triple 5:", triple(5))
```
**Output:**
```text
Double 5: 10
Triple 5: 15
```

---

## 5. Lifetime of Variables

- **Local variables:** Created when the function starts execution and destroyed (garbage collected) as soon as the function returns/exits.
- **Global variables:** Created when the module/script is loaded and exist until the execution of the program terminates.
