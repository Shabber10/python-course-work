# 3. Scope and Lifetime of Variables

Not all variables in a Python program are accessible from everywhere. Where a variable is defined determines its **scope** (visibility) and **lifetime** (how long it exists in memory).

---

## 1. Local Scope
A variable created *inside* a function belongs to the local scope of that function, and can only be used inside that function.

**Input:**
```python
def my_func():
    x = 300  # Local variable
    print("Inside function:", x)

my_func()

# Trying to access x outside the function will raise a NameError!
try:
    print(x)
except NameError as e:
    print("Error:", e)
```
**Output:**
```text
Inside function: 300
Error: name 'x' is not defined
```

---

## 2. Global Scope
A variable created in the main body of the Python code is a **global variable** and belongs to the global scope. Global variables are available from within any scope, global and local.

**Input:**
```python
x = 300  # Global variable

def my_func():
    # We can read global variables directly!
    print("Reading global x inside function:", x)

my_func()
print("Reading global x outside function:", x)
```
**Output:**
```text
Reading global x inside function: 300
Reading global x outside function: 300
```

---

## 3. The `global` Keyword
If you need to **modify** a global variable from inside a function, you must use the `global` keyword. Otherwise, Python will treat it as a new local variable with the same name.

### Modifying without `global` (Creates a local shadow):
**Input:**
```python
x = 10

def change_val():
    x = 20  # This creates a NEW local variable 'x' and does NOT modify global 'x'
    print("Inside function:", x)

change_val()
print("Outside function:", x)
```
**Output:**
```text
Inside function: 20
Outside function: 10
```

### Modifying with `global`:
**Input:**
```python
x = 10

def change_val():
    global x  # Declares that we want to use the global 'x'
    x = 20
    print("Inside function (modified global):", x)

change_val()
print("Outside function:", x)
```
**Output:**
```text
Inside function (modified global): 20
Outside function: 20
```

---

## 4. Nested Functions and `nonlocal`
If you define a function inside another function (nested function), the inner function can read variables from the outer function's scope. If you want to modify a variable in the outer function's scope, use the `nonlocal` keyword.

**Input:**
```python
def outer_func():
    msg = "hello"
    
    def inner_func():
        nonlocal msg  # Declares that we want to modify the variable in the nearest outer scope
        msg = "hello updated"
        
    inner_func()
    print("Outer function message:", msg)

outer_func()
```
**Output:**
```text
Outer function message: hello updated
```

---

## 5. Lifetime of Variables
- **Local variables:** Created when the function starts execution and destroyed (garbage collected) as soon as the function returns/exits.
- **Global variables:** Created when the module/script is loaded and exist until the execution of the program terminates.
