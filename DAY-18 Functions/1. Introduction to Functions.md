# 1. Introduction to Functions

A **function** is a block of organized, reusable code that is used to perform a single, related action. Functions provide better modularity for your application and a high degree of code reusing.

---

## 1. Defining and Calling a Function
In Python, functions are defined using the `def` keyword, followed by the function name, parentheses `()`, and a colon `:`. The code block inside the function is indented.

### Syntax:
```python
def function_name(parameters):
    """
    Optional Docstring (documentation of what the function does)
    """
    # function body code block
    return value
```

### Basic Example:
**Input:**
```python
# Defining a simple function
def greet():
    print("Hello! Welcome to the Python Masterclass.")

# Calling the function
greet()
greet()
```
**Output:**
```text
Hello! Welcome to the Python Masterclass.
Hello! Welcome to the Python Masterclass.
```

---

## 2. Under the Hood: Call Stack & Stack Frames
When you call a function, Python suspends the current flow of execution and allocates a block of memory called a **Stack Frame** (or Activation Record) on the **Call Stack** to handle the function call.

### 📍 How Python Executes a Function
1. **Pushes** the function's Stack Frame onto the Call Stack. This frame holds the function's local variables and arguments.
2. **Executes** the code block inside the function body.
3. **Pops** the Stack Frame off the Call Stack once the function encounters a `return` or finishes running. This returns execution back to the caller script and frees the memory.

```text
CALLING: main script calls greet()
[ Stack Frame: greet() ]  --> Pushed onto stack
[ Stack Frame: main    ]

RETURNING: greet() finishes
[ Stack Frame: main    ]  --> Frame popped, returns to main script
```

---

## 3. Why Do We Need Functions?
1. **Code Reusability:** Write once, run many times. Avoid repeating code (DRY - Don't Repeat Yourself).
2. **Modularity:** Break a large project into smaller, manageable chunks.
3. **Readability:** Naming a block of code (e.g. `calculate_tax()`) makes it clear what the code does.
4. **Easier Debugging:** If a bug exists in a function, you only need to fix it in one place.

---

## 4. Function Docstrings
A **docstring** (documentation string) is a string literal that occurs as the very first statement in a function definition. It is used to describe what a function does. It is surrounded by triple quotes (`"""`).

You can access the docstring of a function using the `__doc__` attribute or the `help()` function.

**Input:**
```python
def add_numbers(a, b):
    """
    This function takes two numbers as inputs and returns their sum.
    """
    return a + b

# Printing the docstring
print("Docstring using __doc__:")
print(add_numbers.__doc__)

print("Calling help():")
help(add_numbers)
```
**Output:**
```text
Docstring using __doc__:

    This function takes two numbers as inputs and returns their sum.
    
Calling help():
Help on function add_numbers in module __main__:

add_numbers(a, b)
    This function takes two numbers as inputs and returns their sum.
```
