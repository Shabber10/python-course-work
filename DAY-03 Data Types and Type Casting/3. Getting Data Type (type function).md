# Getting Data Type (type function)

You can instantly check the exact data type of any object or variable by using the built-in `type()` function. This is incredibly useful for debugging.

---

## 1. Using the `type()` Function

**Input:**
```python
x = 5
y = "Hello World"
is_active = True
fruits = ["apple", "banana"]

print(type(x))
print(type(y))
print(type(is_active))
print(type(fruits))
```
**Output:**
```text
<class 'int'>
<class 'str'>
<class 'bool'>
<class 'list'>
```

## 2. Dynamic Typing and Object References

In Python, **data type information is associated with the object itself in memory, not the variable name**. The variable is merely a pointer or label referencing the object.

When you run:
```python
x = 500       # x references an integer object 500
x = 'Data'    # x is re-bound to reference a string object 'Data'
```
1. Python creates the integer object `500` in heap memory and points `x` to it.
2. When `x = 'Data'` is executed, Python creates a new string object `'Data'`, points `x` to it, and drops the reference to `500`. The integer object `500` now has zero references and is cleaned up by the garbage collector.

### Shared References (e.g. `b = a`)
When you assign one variable to another, they both reference (point to) the *same* object in memory.

For example:
```python
a = 3
b = a       # b now points to the same integer object 3 as 'a'
a = 'spam'  # 'a' is re-bound to point to a new string object 'spam'
```
* **Result:** `b` still references the original integer object `3`. It does *not* change to `'spam'` because variables are independent references.
