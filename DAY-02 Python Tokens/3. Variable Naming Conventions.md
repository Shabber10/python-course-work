# 3. Variable Naming Conventions

Because variables can hold any type of data and serve many different purposes, it is absolutely critical to name them correctly. Python has strict technical rules for naming variables, as well as community-accepted "Best Practices" for how to format them.

---

## 1. The Strict Rules of Naming
If you break any of these rules, your Python program will crash with a `SyntaxError`.

1. **A variable name MUST start with a letter or the underscore character (`_`).**
2. **A variable name CANNOT start with a number.**
3. **A variable name can only contain alpha-numeric characters and underscores (`A-z`, `0-9`, and `_`). No spaces or special symbols like `@` or `!` are allowed.**
4. **Variable names are case-sensitive.** (`age`, `Age` and `AGE` are three completely different variables in memory).

**Input:**
```python
# VALID Variable Names
myvar = "John"
my_var = "John"
_my_var = "John"
myVar = "John"
MYVAR = "John"
myvar2 = "John"

# INVALID Variable Names (These will cause errors)
# 2myvar = "John"    -> Starts with a number
# my-var = "John"    -> Contains a hyphen
# my var = "John"    -> Contains a space
```

## 2. Multi-Word Variable Formatting
Variable names with more than one word can be incredibly difficult to read if you don't use a formatting convention. 

**Input:**
```python
# Hard to read
numberofapples = 50 
```

To make multi-word variables readable, Python developers use three main formatting styles:

### A. Camel Case
Each word, **except the first**, starts with a capital letter. This is very popular in JavaScript, but sometimes used in Python.
**Input:**
```python
myVariableName = "John"
numberOfApples = 50
```

### B. Pascal Case
**Every word** starts with a capital letter. In Python, this is strictly reserved for naming **Classes** (which you will learn about in Object Oriented Programming).
**Input:**
```python
MyVariableName = "John"
NumberOfApples = 50
```

### C. Snake Case (PEP 8 Recommended)
Each word is separated by an underscore character. **This is the official, community-standard way to name variables in Python.**
**Input:**
```python
my_variable_name = "John"
number_of_apples = 50
```

## 3. Best Practices for Developers
1. **Be Descriptive:** A variable named `a = 15` is terrible because nobody knows what `a` means. A variable named `user_age = 15` is perfect.
2. **Use English:** Since most programming languages use English keywords (`if`, `for`, `while`), it is standard practice to name variables in English too.
3. **Avoid Reserved Keywords:** Never name a variable after a built-in Python function or keyword.
   * *Bad:* `print = 5` (You just overwrote the print function!)
   * *Bad:* `list = [1, 2, 3]` (You just overwrote the built-in list type!)
